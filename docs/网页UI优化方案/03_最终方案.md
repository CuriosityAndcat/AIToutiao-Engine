# 网页 UI 优化方案 · Step 3 最终方案（经 Step 2 调研收敛 + Step 4 自审）

> 本方案是本轮「用户对 UI 整体不满意」的正式重设计计划。
> 执行范围：**Track 1+2+3（纯 UI 层，零生产逻辑改动，零/低新依赖）**；Track 4 拆分单列，不在本轮自动执行（Tier 3）。
> 所有改动限定 `engine_app.py` 的 UI 部分。

---

## 设计原则
1. 原生优先：用 `st.tabs` / `st.dataframe` / `st.status` / `st.container` 等原生组件，零新增硬依赖。
2. Token 化：补全 `:root`（spacing/radius/typography），消除 inline `style=` 硬编码颜色。
3. 分区不堆叠：用顶部 Tab 解决单页长滚动；成果区整宽展示质检，稿件/配图合理分栏。
4. 安全/一致性：保留上一轮已修项（魔法数字/孤儿键/色盲标签/响应式），不回退。

---

## Track 1 — 信息架构重组（最高回报）
- **T1.1 顶部导航 Tab**：`st.tabs(["🚀 运行监控", "📄 成果展示", "⚙️ 配置"])`。
  - 运行监控：URL 输入 + 阶段灯 + 进度条 + 日志（运行中 fragment 自动刷新落在此 tab）。
  - 成果展示：质检仪表盘 + 稿件 + 配图 + 文件清单（完成态）。
  - 配置：将侧栏配置迁入此 tab（或保留侧栏——见 T1.3 决策）。
- **T1.2 成果区布局重构**：
  - 质检仪表盘改为 `st.dataframe`（可横向滚动），**独占整宽**，不再塞进 1/3 列 expander（解决 A2）。
  - 稿件与配图：`col_img(1):col_text(2)` 改为「稿件卡（主）+ 配图/统计（侧）」或上下分区，确保可读。
- **T1.3 侧栏决策**：默认**保留侧栏**放核心开关（风格/类型/人工化/配图），把「API 密钥 / 下载设置 / 快捷参考」收进「配置」tab 的 expander，减轻侧栏臃肿。

## Track 2 — 视觉层级升级
- **T2.1 Token 补全**：`:root` 增加 `--space-1..5`(4/8/12/16/24)、`--radius-sm/md/lg`(6/8/12)、`--font-mono/sans`；`_inject_css` 引用；`render_main` 的 `status_color` 字符串 → 改为 `.status-*` class 引用（消除 B6/B12）。
- **T2.2 卡片化与阴影**：`.result-card`/`.log-container`/新增 `.panel` 加 `box-shadow` 与更细 border；阶段灯之间加连接线，形成 step 进度视觉（解决 B5）。
- **T2.3 进度条加阶段刻度**：在 `st.progress` 上方渲染 5 段刻度锚点，与阶段灯对齐，去掉信息重复感（解决 B7）。

## Track 3 — 交互增强
- **T3.1 运行态闪烁修复**：fragment 内仅刷新「日志容器 + 阶段灯 + 下一步按钮」，不重绘整页；或稳定 `key` 防止标题/URL 区重建。
- **T3.2 首启阶段感**：`st.spinner` → `st.status`，分阶段文案（"加载 PyTorch…"→"加载模型…"→"就绪"）（解决 C8）。
- **T3.3 结果操作按钮响应式**：`columns(4)` 在窄屏（`@media max-width:600px`）降为 `columns(2)` 或纵向堆叠（解决 C9）。

## Track 4 — 可维护性（Tier 3，单独立项，本轮不执行）
- **T4.1 拆分** `engine_app.py`(1967 行) → `ui/`（`css.py`/`sidebar.py`/`main.py`/`progress.py`/`logs.py`/`results.py`）+ `app.py` 入口 + 编排层保留。
- 对应 AGENTS.md E-3；需先出详细 Plan 与依赖校验（参考 `code-splitter`）。

## 执行顺序（Step 5 计划）
1. `_inject_css`：补 token + 卡片阴影 + 阶段连接线 + 刻度样式。
2. `main()`：包 `st.tabs`，分流「运行监控/成果展示/配置」；fragment 落在运行监控 tab。
3. `render_results`：质检改 `st.dataframe` 整宽；稿件/配图重排版。
4. `render_main`/`render_progress`：去 inline 硬编码颜色，引用 token class；加阶段刻度。
5. `main()` 首启：换 `st.status` 分阶段。
6. 响应式收尾：结果操作按钮窄屏适配。

## 验收锚点（详见 04_终验方案）
- 静态：`ast.parse` 通过；无新增 inline `style=` 颜色硬编码；`st.tabs` 三 Tab 存在；质检用 `st.dataframe`；`:root` 含 `--space/--radius` token。
- 视觉：信息架构由「中下」→「中上」；运行态无 3s 全页闪烁。
- 5 维终评目标：≥ 90。
