# 文章产出强制规范（Article Output Specification）

> **版本**：v1.1-draft（草稿，新增第5章配图规范，待评审）
> **定位**：AIToutiao-Engine 所有文章/微头条产出的**唯一强制规范**。本文覆盖「结构 → 语言 → 事实 → 门禁 → 配图」五层，是 `evaluation.py` / `humanize.py` / `prompts/` / `_red_lines.py` / `engine_app.py` 配图链路的上游约束。
>
> **使用方式**：本规范在以下环节生效：
> - **写作阶段**：注入 LLM System Prompt 作为格式/结构约束
> - **humanize 阶段**：作为改写目标标准
> - **评估阶段**：`evaluation.py` 的 5 维评分基准
> - **发布前**：自动化检查清单的判定依据
>
> **修订纪律**：任何对文章产出标准的修改必须先更新本文，再同步到下游代码。

## 1. 文章结构与格式（强制）

> **设计者**：writing-prompt-engineer（写作提示词工程师）
> **覆盖范围**：字数/标题/钩子/正文结构/结尾/配图/语言铁律/平台规则

### 1.1 文章类型与字数约束

#### 1.1.1 微头条（`ContentType.TOUTIE`）

| 参数 | 约束 | 级别 |
|------|------|------|
| 正文字数 | **800–1200 字**（含标点，不含标题与话题标签） | **[强制]** |
| 标题字数 | **≤30 字** | **[强制]** |
| 段落数 | **≥6 段**，每段不超过 200 字 | **[强制]** |
| 配图 | 封面 1 张（必选），内文 0–2 张（可选） | **[强制]** |

> **对齐说明**：`models.py` 中 `TOUTIE` 当前注释标注 800–1200 字，本规范确认此范围为硬约束。`humanize.py` 要求「段落翻倍」后，原始段落数下限为 3，翻倍后 ≥6 段。

#### 1.1.2 文章（`ContentType.ARTICLE`）

| 参数 | 约束 | 级别 |
|------|------|------|
| 正文字数 | **1800–2500 字**（含标点，不含标题与话题标签） | **[强制]** |
| 标题字数 | **≤40 字** | **[强制]** |
| 章节数 | **≥3 个一级章节**，每章 ≥2 段 | **[强制]** |
| 配图 | 封面 1 张（必选），内文每章 ≥1 张 | **[强制]** |

> **对齐说明**：`models.py` 中 `ARTICLE` 当前注释约 2000 字，本规范细化为 1800–2500 字区间。下限保证信息完整（`evaluation.py` 信息完整维度），上限防止 AI 灌水。

#### 1.1.3 字数计数规则 **[强制]**

- 以 `len(re.sub(r'\s+', '', text))` 计中文字符数（去掉所有空白后计算长度）。
- 标点符号计入字数。
- 话题标签（`#xxx#`）、引用块（`> ` 开头行）不计入正文字数。
- 标题单独计数，不计入正文。

### 1.2 必需要素（缺一不可）

#### 1.2.1 标题

**通用标题铁律 [强制]**

| 规则 | 说明 |
|------|------|
| 必须包含**具体信息点** | 禁止纯感叹/纯疑问/纯悬念（如"这件事太震撼了！"） |
| 禁止标题党 | 标题承诺的内容必须在正文前 1/3 处兑现 |
| 禁止使用省略号（…/...） | 用问号、冒号、破折号替代 |
| 字符级去 AI 味 | 禁止模板标题格式：`A：B，C 背后 D` |

**各风格标题公式 [强制]**

| 风格 | 标题公式 | 示例 | 禁用 |
|------|----------|------|------|
| **包明说** | `[大国/战略主体] + [动作/态势] + [？] + [反差结论]` | `俄罗斯这一手，把北约算死了？三大战略通道全部锁死` | `深度分析/全面解读/重磅` 等营销词 |
| **晋说** | `[数据] + [事实判断] + [!]` | `30万亿国债，6.5%增速——这盘棋，美国看懂了！` | 模糊形容词（`震撼/惊人/炸裂`） |
| **全球档案馆** | `[国家/地区] + [事件代号/时间戳] + [：] + [核心档案标签]` | `乌克兰 2024：三份被忽略的战报，一份未公开的调令` | 情绪化标点叠加（`？！`、`！！！`） |
| **听风的蚕** | `[第一人称视角/代号] + [反常现象] + [？]` | `我在也门拍到的那架无人机，编号不该出现在那里` | 第三人称客观陈述（`据悉/据分析/数据显示`） |

#### 1.2.2 钩子段（正文第一段）

**通用规则 [强制]**

| 规则 | 级别 |
|------|------|
| 第一句 **≤20 字** | **[强制]** |
| 禁止以"近日""据报道""据悉""众所周知"开头 | **[强制]** |
| 禁止以背景介绍/名词解释开头 | **[强制]** |
| 必须在 3 句内制造**认知落差**（事实 vs 常识、预期 vs 现实） | **[强制]** |

**钩子段四种推荐模式 [建议]**

1. **场景切入式**：一个具体的时间+地点+动作，直接拉入现场。
2. **反常识断言式**：一个与大众认知相反的判断，紧随一句话证据。
3. **悬念叙事式**：一个未完成的动作/事件，制造追问冲动。
4. **金句引爆式**：一句独立成段的判断句，自带传播力。

#### 1.2.3 正文分层结构

**微头条结构模板 [强制]**

```
[钩子段 1 段]
    ↓
[核心事实展开 2–3 段]  ← 必须包含至少 1 处具体数据/引述
    ↓
[观点/分析 2–3 段]     ← 必须包含至少 1 处金句
    ↓
[收尾段 1 段]          ← 见 1.2.4
```

**文章结构模板 [强制]**

```
[钩子段 1–2 段]
    ↓
第一章：背景/脉络（≥2 段）
    ↓
第二章：核心事件/数据拆解（≥3 段）← 每段 ≤200 字
    ↓
第三章：趋势研判/深层逻辑（≥2 段）
    ↓
[收尾段 1–2 段]
```

> **对齐说明**：包明说风格 System Prompt 中的"7 层递进"结构（现象→数据→历史→对比→逻辑→趋势→收尾）作为文章类型的高级选项，不强制所有风格套用。

**段落约束 [强制]**

| 规则 | 微头条 | 文章 |
|------|--------|------|
| 单段最大字数 | 200 字 | 250 字 |
| 超短句（≤15 字独立成段） | ≥3 处 | ≥5 处 |
| 金句（独立成段的判断句） | ≥3 处 | ≥5 处 |
| 数据引用 | ≥1 处 | ≥3 处 |

> **对齐说明**：超短句与金句要求与 `humanize.py` 中 `SHORT_SENTENCE_MIN=3`、`GOLDEN_QUOTE_MIN=3` 对齐。ARTICLE 类型因篇幅更长，各指标上浮。

#### 1.2.4 结尾

**通用规则 [强制]**

- 禁止"综上所述""总而言之""通过以上分析可以看出"等模板化收束语。
- 禁止新信息出现在最后一段（不得在结尾引入前文未提及的事实/数据）。
- 结尾必须包含**明确的交互钩子**。

**各风格结尾模式 [强制]**

| 风格 | 结尾模式 | 示例 |
|------|----------|------|
| 包明说 | 反向式收束 + 战略视角升华 | `当三条通道全部锁死，北约的东扩还剩下什么？` |
| 晋说 | 数据锚定 + 趋势判断 | `30万亿不是终点，6.5%才是变量——剩下的，交给时间。` |
| 全球档案馆 | 档案标签 + 开放悬置 + 互动号召 | `这份调令的落款日期是2024年3月。在那之后72小时发生了什么——我们下期继续。家人们，你们怎么看？评论区见。` |
| 听风的蚕 | 个人落款 + 留扣追问 | `那架无人机的编号我查了三天。如果你知道什么，评论区告诉我。咱且看，下回分解。` |

#### 1.2.5 配图规范（插入规则）

> **详细配图质量规范见第5章**。本节仅定义图片在正文中的**插入位置**规则。

| 规则 | 级别 |
|------|------|
| 封面图必须在正文开始前插入（`![封面](images/cover.png)` 或平台封面字段） | **[强制]** |
| 微头条内文图 0–2 张，文章每章 ≥1 张（详见 §5.1.1） | **[强制]** |
| 内文图必须紧跟相关段落之后，不得插入段落中间打断阅读流 | **[强制]** |
| 图片 alt 文本必须包含具体描述，禁止 `图1`、`配图` 等无意义标签 | **[强制]** |
| 禁止使用带水印/来源不明的图片 URL | **[建议]** |
| 组装前必须校验图片文件存在（`Path.exists()`），缺失则跳过注入不容错 | **[强制]** |

### 1.3 语言铁律（跨风格通用）

#### 1.3.1 句长限制

| 风格 | 单句上限 | 级别 |
|------|----------|------|
| 包明说 | **≤15 字** | **[强制]** |
| 晋说 | **≤25 字**（数据句 ≤30 字） | **[强制]** |
| 全球档案馆 | **≤25 字** | **[强制]** |
| 听风的蚕 | **≤20 字** | **[强制]** |

**超长句例外**（仅以下情形可突破上限）：
- 直接引语（引用人物原话）
- 数据罗列句（如"GDP 增速分别为 3.2%、4.1%、5.0%"）

#### 1.3.2 禁用词清单 [强制]

**模板化结构词（100% 禁用）**
```
首先、其次、再次、最后
第一、第二、第三
一方面、另一方面
总而言之、综上所述、总的来看、通过以上分析可以看出
值得注意的是、值得一提的是、需要强调的是
众所周知、不言而喻、显而易见
究其原因、其背后原因、究其根本
```

**AI 味修饰词（100% 禁用）**
```
深度、重磅、独家（除非确为独家）
震撼、惊人、炸裂、逆天
这一、这一事件、这一举措、这一决定
无疑、必将、势必
在……背景下、在……框架下、在……指引下
不仅……而且……（连续使用）
既……又……（连续使用）
```

**新闻腔词汇（100% 禁用）**
```
近日、日前、据悉、据相关媒体报道
据报道、据知情人士透露
记者了解到、记者从……获悉
```

**语气词限制 [强制]**

| 规则 | 级别 |
|------|------|
| "呢/吧/吗/啊/嘛" 等口语语气词：每 150 字至少出现 1 处 | **[强制]** |
| 同一语气词不得连续两句使用 | **[强制]** |

> **对齐说明**：与 `humanize.py` 中 `TONE_WORD_INTERVAL=150` 对齐。

#### 1.3.3 书面语→口语映射表 [强制]

| 书面语（禁用） | 口语替代（必须） |
|---------------|------------------|
| 该（指代） | 这个 / 这次 |
| 其 | 它的 / 他的 |
| 已（完成态） | 已经 / 了 |
| 未（否定态） | 没有 / 还没 |
| 若 | 如果 |
| 即 | 就是 |
| 均 | 都 |
| 仅 | 只 |
| 进行 | 做 / 干 / 搞（按语境） |
| 予以 | 给 |
| 位于 | 在 |
| 呈现出 | 显出 / 表现出来 |
| 之所以……是因为 | 之所以……就因为 |
| 在……当中 | 在……里 |
| ……之一 | 一个…… |
| 具有……的意义 | 意味着…… / 标志着…… |
| 呈现……态势/趋势 | 正在…… / 越来越…… |

#### 1.3.4 标点与格式禁令 [强制]

| 规则 | 级别 |
|------|------|
| 禁止 `？！` 叠加（仅可单独使用 `？` 或 `！`） | **[强制]** |
| 禁止 `！！！` 多感叹号叠加 | **[强制]** |
| 禁止 `。。。。。` 等重复标点 | **[强制]** |
| 禁止 emoji：正文 0 emoji（标题最多 1 个） | **[强制]** |
| 禁止全角英文/数字（中文语境下英文和数字使用半角） | **[强制]** |
| 禁止连续 3 句以上以同一标点结尾 | **[建议]** |
| 引号内引用他人原话须用中文双引号 `""`，不得用 `""` | **[强制]** |

#### 1.3.5 人称与视角 [强制]

| 风格 | 人称要求 |
|------|----------|
| 包明说 | 第三人称为主，分析段可插入"我们"，禁止"笔者""本文" |
| 晋说 | 第三人称，数据段可插入"我们看到"，禁止"笔者认为" |
| 全球档案馆 | 第一人称复数"我们"（机构视角），禁止"我""笔者" |
| 听风的蚕 | 第一人称"我"（个人视角），强制使用 |

## 2. 去AI味与语言质量（强制）

> **设计者**：content-humanizer（去AI味润色编辑）
> **覆盖范围**：可自动化检测硬指标 / LLM 评估软指标 / 分风格差异化 / 验收门禁

### 2.1 可自动化检测的硬指标 [强制]

以下指标不依赖 LLM 主观判断，可通过规则/正则/统计直接判定。所有检测应在 `evaluation.py` 第 5 维（"去AI味"）LLM 评分**之前**运行，作为硬门槛：任一硬指标不达标 → 整篇判 `NEEDS_IMPROVEMENT`，不再进入 LLM 评分。

#### 2.1.1 禁用词/禁用句式检测

| 检测项 | 检测方法 | 阈值 | 判定 |
|---|---|---|---|
| **AI 味过渡词**：首先/其次/最后/综上所述/总而言之/值得注意的是/需要指出的是/与此同时/另一方面/此外/总之 | 正则匹配，统计命中次数 | 命中数 > 0 | 不通过 |
| **AI 味句式**：`从……角度来看` / `在……的背景下` / `随着……的发展` / `不可否认的是` / `毋庸置疑` | 正则匹配 | 命中数 > 0 | 不通过 |
| **微头条禁用"近日/据悉/据外媒报道"起头**（包明说/晋说/听风的蚕风格） | 正文首 30 字符内正则匹配 `^(近日\|据悉\|据.*报道)` | 命中 → 不通过 |
| **全球档案馆禁止吐槽标记**：`说实话/就离谱/我服了/真无语了/这操作我是真没看懂` | 正则匹配 | 命中数 > 0 | 不通过 |
| **全球档案馆禁止 #话题标签** | 正文末段正则匹配 `#\S+` | 命中 → 不通过 |

#### 2.1.2 句长分布检测

| 检测项 | 检测方法 | 阈值 | 判定 |
|---|---|---|---|
| **超短句（2-5 字）数量** | 以 `。！？` 为分隔符拆分句子，统计字数 ∈ [2,5] 的句数 | ≥ 3 处 | 不达标 → 不通过 |
| **长句（30+ 字）数量** | 同上，统计字数 ≥ 30 的句数 | ≥ 2 处 | 不达标 → 不通过 |
| **句首重复度** | 提取相邻句子首词（取前 2 个字符），统计重复率 | 相邻句首重复率 ≤ 10% | 超标 → 不通过 |
| **平均句长** | 总字数 / 句子数 | 12-22 字（微头条通用）；全球档案馆 15-25 字 | 偏离 → 降级为 [建议]（不硬卡，因风格差异大） |

> **注意**：句长检测需排除纯数字/英文/URL 行。超短句定义为"包含至少 1 个中文字符且总字数 2-5"。

#### 2.1.3 段落结构检测

| 检测项 | 检测方法 | 阈值 | 判定 |
|---|---|---|---|
| **段落数比例**（微头条通用/包明说/晋说/听风的蚕） | `输出段数 / 输入段数` | ≥ 2.0 | 不达标 → 不通过 |
| **段落长度标准差** | 统计各段落字数，计算标准差 | 标准差 ≥ 段落平均字数的 30%（即明显不均衡） | 不达标 → 不通过 |
| **全球档案馆：Markdown 章节标题保留** | 检测输出中 `## ` 开头的行数 | 输出 ≥ 输入章节数 | 缺失 → 不通过 |
| **微头条通用：结尾话题标签** | 末段正则匹配 `#\S+` | ≥ 3 个 | 不足 → 不通过 |

#### 2.1.4 语气词与态度标记密度检测

| 检测项 | 检测方法 | 阈值 | 判定 |
|---|---|---|---|
| **语气词密度**（微头条通用/包明说/晋说/听风的蚕） | 正则匹配 `[啊呢吧嘛哈咯呗罢了]`；`密度 = 命中数 / (总字数 / 150)` | 密度 ≥ 1.0 | 不达标 → 不通过 |
| **个人态度标记数量**（微头条通用/包明说/晋说/听风的蚕） | 正则匹配 `说实话\|讲道理\|我跟你讲\|不吹不黑\|我是真觉得\|我个人看法\|我反正\|就离谱\|我服了\|这操作` | ≥ 3 处 | 不达标 → 不通过 |
| **全球档案馆语气词上限** | 同上正则 | 密度 ≤ 0.5（即每 300 字 ≤ 1 个） | 超标 → 不通过 |
| **全球档案馆馆长叙事标记** | 正则匹配 `馆长跟家人们说\|有小伙问馆长\|馆长当年\|馆长我` | ≥ 2 处 | 不达标 → 不通过 |
| **听风的蚕方言词** | 正则匹配 `中！\|咋整\|恁\|这家伙\|俺\|得劲\|不瓤\|乖乖\|了不得了\|拉倒吧\|中中中\|弄啥嘞` | ≥ 3 处 | 不达标 → 不通过 |
| **听风的蚕评书话术** | 正则匹配 `咱今儿个\|您猜怎么着\|话说这一天\|咱书接上文\|列位看官\|说到这儿\|恁可能要问了\|这事儿说来话长\|您猜这里头\|真相是啥呢\|重点来了啊\|高啊\|中！这家伙\|咱且看\|下回分解` | ≥ 每段 1 种 | 不达标 → 不通过 |

#### 2.1.5 书面→口语替换率检测

| 检测项 | 检测方法 | 阈值 | 判定 |
|---|---|---|---|
| **书面词残留** | 正则匹配 `进行\|实施\|呈现\|指出\|充分\|显著\|极其\|予以\|加以\|致力于`（在非引用/非固定搭配语境） | 每 100 字命中数 ≤ 1 | 超标 → 不通过 |
| **口语连接词多样性** | 正则匹配 `然后呢\|再一个\|关键是\|说白了\|你想想\|这么说吧\|咱就说\|反正\|就是说\|老实讲\|怎么说呢\|你再琢磨琢磨\|说到根上`，统计不同种类数 | ≥ 5 种 | 不足 → 不通过 |

#### 2.1.6 硬指标一票否决清单汇总

以下 10 项中**任一不达标即判整篇不通过**（在 LLM 评估之前执行）：

1. AI 味过渡词残留 > 0
2. AI 味句式命中 > 0
3. 超短句（2-5字）< 3 处
4. 句首相邻重复率 > 10%
5. 段落数比例 < 2.0（全球档案馆除外，见 2.3）
6. 段落长度标准差 < 平均段长的 30%
7. 语气词密度 < 1.0（全球档案馆除外，见 2.3）
8. 个人态度标记 < 3 处（全球档案馆/听风的蚕除外，见 2.3）
9. 微头条禁用"近日/据悉"起头（适用风格）→ 一票否决
10. 分风格专属硬指标（见 2.3 表格）

### 2.2 需 LLM 评估的软指标 [强制]

以下指标无法通过规则/正则可靠判定，必须由 LLM 评估（即 `evaluation.py` 第 5 维「去AI味」的评分依据）。

#### 2.2.1 「像真人写作」综合判断

**评估要点**：
- **非书面报告感**：是否像一个人在说话/讲故事，而非机关简报或学术论文
- **节奏自然度**：长短句交错是否自然（非机械地"每 3 句插 1 个短句"）
- **废话/闲笔的适度存在**：真人写作偶尔会绕个弯、插一句无关紧要的话再拉回来。完全"字字有用"是典型 AI 特征
- **情绪真实波动**：是否有真实的情绪起伏，情绪转折是否有铺垫

**评分指引**：
- 90-100：读完全文无法判断是 AI 还是真人，有多处"人味细节"
- 75-89：整体像人写的，但个别段落仍有"模板感"
- 60-74：能明显感觉到机器腔，但有一些人性注入
- <60：一眼 AI，需重写

#### 2.2.2 个人态度标记的自然度

- 态度标记是否"嵌"在叙事中，而非机械堆砌
- 态度标记是否有变化（震惊/质疑/吐槽/释然），而非同一情绪重复 3 次
- 态度标记是否符合语境

#### 2.2.3 情绪曲线合理性（分风格）

- **微头条通用/包明说/晋说**：好奇→震惊→讽刺/愤怒→升华，单调递进
- **全球档案馆**：震惊→紧张→愤怒→自豪→升华，**必须单调上升，禁止回拉**
- **听风的蚕**：评书节奏——铺垫→设悬念→揭秘→喝彩→留扣，波浪感

#### 2.2.4 AI Tell 集群检测（防误判）

源自 humanizer 方法论核心原则：**孤立特征不算 AI tell，集群才算**。

LLM 评估时必须执行：
1. 列出所有可疑的 AI 特征（孤立点）
2. 判断它们是否形成集群（≥ 3 个特征同时出现在同一段落/相邻段落）
3. 排除人类写作特征（具体不易伪造的细节、矛盾情绪、时代梗、句长变化、真实插话自我纠正）
4. 仅对确认的 AI tell 集群做扣分

### 2.3 分风格差异化标准 [强制]

| 维度 | 包明说 | 晋说 | 全球档案馆 | 听风的蚕 |
|------|--------|------|-----------|----------|
| **核心去AI味方向** | 短句机枪感 + 冷峻讽刺 + 数据佐证 | 口语吐槽 + 典故细节 + 乡愁升华 | 馆长口吻 + 情绪单调上升 + 金句密度 | 评书话术 + 方言底色 + 留扣收尾 |
| **段落数比例** | ≥ 2.0 | ≥ 2.0 | **不适用**（保留章节结构） | ≥ 2.0 |
| **平均句长** | ≤ 15 字（硬指标） | 12-20 字 | 15-25 字 | 15-22 字 |
| **个人态度标记** | ≥ 3 处（冷峻型：`不吹不黑/问题在于/你仔细想想`） | ≥ 3 处（吐槽型：`最离谱的是/所以别再被忽悠了/你确定？肯定？以及一定？`） | **禁止** `说实话/就离谱/我服了`（用金句+馆长叙事替代） | ≥ 3 处（评书型：`俺服了/中！这家伙/乖乖，了不得了`） |
| **语气词密度** | 每 150 字 ≥ 1 个 | 每 150 字 ≥ 1 个 | 每 300 字 ≤ 1 个（**上限**，非下限） | 每 150 字 ≥ 1 个（含方言语气词） |
| **专属硬指标** | 句首不重复率 > 90%；句子平均 ≤ 15 字 | 反问句 ≥ 2 处（`？`结尾句占比 ≥ 10%） | 金句 ≥ 3 处且来自 ≥ 3 种模板；馆长叙事 ≥ 2 处；情绪单调上升（无回拉）；结尾为金句+互动号召（非#话题标签） | 方言词 ≥ 3 处；评书话术命中数 ≥ 段落数；结尾为留扣 |

### 2.4 验收门禁设计

#### 2.4.1 两层门禁流程

```
产出文本
  │
  ├── 第 0 层：硬指标预检（2.1 节全部规则）← 纯代码，不调 LLM
  │     │
  │     ├── 通过 → 进入第 1 层
  │     └── 不通过 → 直接判 NEEDS_IMPROVEMENT
  │           └── 返回具体不达标项清单
  │
  └── 第 1 层：LLM 5 维评分（现有 evaluation.py 流程）
        │
        ├── 第 5 维「去AI味」评分 < 50 → 不通过
        ├── 综合分 < threshold（通用 75 / 研究写作 80） → 不通过
        ├── 任一项 < 50 → 不通过
        ├── 事实准确性 < FACT_HARD_FLOOR(80) → 不通过
        └── 全部达标 → PASS
```

#### 2.4.2 自愈闭环集成

当第 0 层或第 1 层判定不通过时，反馈信息应结构化：

```python
# 硬指标不通过时的反馈结构（示例）
{
  "stage": "humanize_hard_gate",
  "passed": false,
  "failures": [
    {"rule": "超短句数量", "current": 1, "required": "≥ 3"},
    {"rule": "禁用词残留", "hits": ["与此同时", "值得注意的是"]},
  ]
}
```

这样 `write_stage.py` 可以将具体失败原因注入重写 prompt，形成精准自愈而非盲目重试。

#### 2.4.3 人类写作特征保护（防误判）

以下特征**不应**被判定为 AI tell：
- 具体不易伪造的细节（如"2023 年 7 月 14 日下午 3 点，北京西站北广场"）
- 矛盾情绪（如"又气又好笑"）
- 时代梗/网络用语（如"这波属于是……"）
- 句长剧烈变化（一句 3 字，下一句 45 字）——这是人味，不是 bug
- 真实插话/自我纠正（如"不对，准确地说应该是……"）

## 3. 内容质量与事实准确性（强制）

> **设计者**：news-briefing-expert（选题情报官）
> **覆盖范围**：信息来源等级 / 事实断言规则 / 内容覆盖度 / 合规红线 / 事实核查工作流

### 3.1 信息来源等级

#### 3.1.1 一级来源：可直接引用，无需额外标注 [强制]

以下来源的信息在正文中可直接陈述为事实，无需附加「据报道」「据分析」等标记：

| 类别 | 国内来源 | 海外来源 |
|------|----------|----------|
| 官方通讯社 | 新华社、中国新闻社 | Reuters、AP News、AFP |
| 政府网站 | 中国政府网、国防部官网、外交部官网、国台办官网 | 联合国官网、各国政府官方声明 |
| 权威军事媒体 | 央视军事、解放军报、中国军网 | Jane's Defence Weekly、IISS |

**约束**：
- 同一事实有多个一级来源报道时，优先使用**发布时间最近**的版本
- 一级来源之间存在矛盾时，**同时保留**并标注分歧，不得选择性引用

#### 3.1.2 二级来源：需显式标注 [强制]

以下来源的信息在正文中使用时，**必须**附加「据报道」「据XX报道」「据公开信息」「据分析」等来源标记：

| 类别 | 国内来源 | 海外来源 |
|------|----------|----------|
| 主流媒体 | 财新网、澎湃新闻、第一财经、观察者网 | BBC、CNN、Bloomberg、Financial Times、Al Jazeera |
| 专业军事分析 | 舰船知识、现代舰船、兵器知识 | RUSI、CSIS、IISS 报告 |
| 行业垂直媒体 | 36氪（官方信源报道）、虎嗅（专家访谈） | TechCrunch、The Verge、Wired |

**约束**：
- 二级来源中的**数据**必须至少有一个一级来源佐证，否则降级标注为「据XX估计」「XX声称」
- 二级来源的**观点性内容**在正文中必须与事实性内容明确区分

#### 3.1.3 禁止来源：绝对不得使用 [强制]

以下来源的信息**禁止**以任何形式进入文章正文：
- 自媒体/个人公众号（无机构背书）
- 贴吧、论坛、知乎个人回答、微博个人账号
- 未经证实的社交媒体帖子（含微信群/QQ群截图、Telegram 频道）
- 匿名来源（"消息人士称"且无机构背书）
- 未注明来源的截图、表格、数据
- AI 生成内容（如 ChatGPT 输出）作为事实来源

**例外**：自媒体/社交媒体的**官方账号**（如@国防部发布 微博、@央视新闻 抖音）视为一级来源。

#### 3.1.4 最低引用来源数 [强制]

| 内容类型 | 最低独立来源数 | 其中一级来源最低数 |
|----------|---------------|-------------------|
| 微头条 | ≥ 2 | ≥ 1 |
| 头条文章 | ≥ 3 | ≥ 1 |
| 公众号长文 | ≥ 4 | ≥ 2 |

「独立来源」指不同机构/媒体的独立报道，同一通讯社被不同媒体转载计为 1 个来源。

---

### 3.2 事实断言规则

#### 3.2.1 可追溯断言 [强制]

以下类型的事实断言**必须**可追溯到至少一个一级或二级来源：
- **数字**：伤亡人数、装备数量、预算金额、时间跨度、距离、百分比
- **日期**：事件发生的具体日期、协议签署日期、部署时间
- **人名**：官员姓名、指挥官姓名、专家姓名（含职务/头衔）
- **地名**：军事基地名称、冲突地点、部署区域
- **武器型号**：具体装备型号（如歼-20、055型驱逐舰、DF-17）

#### 3.2.2 敏感数据强制标注 [强制]

以下类型的数据在正文中出现时，**必须**在句末或段末标注来源：
- 战斗伤亡数字 → 标注「据XX报道」「XX方声称」
- 装备损失数量 → 同上
- 冲突进展描述（"占领""推进""控制"）→ 标注信息来源方
- 协议/协定内容 → 标注「据XX公布的文本」

**示例**：
- ✅ 「据俄国防部 3 月 15 日通报，乌方在库尔斯克方向损失超过 200 人」
- ❌ 「俄军损失坦克 150 辆」（未标注来源方，无法判断数据出自哪一方）

#### 3.2.3 推测性内容标记 [强制]

任何超出已有来源明确记载的推断、分析、预测，**必须**使用以下标记词之一：
- 「可能」「或」「不排除」
- 「据报道」「据公开信息」「据分析」
- 「有观点认为」「某方声称」
- 「如果……那么……」

禁止将推测包装为确定事实。

#### 3.2.4 绝对禁止 [强制]

以下行为**零容忍**：
1. 编造对话：虚构人物之间的对话或引语
2. 编造协议内容：虚构条约条款、协议细节、备忘录内容
3. 编造武器性能数据：虚构或自行推算武器射程/速度/载荷等参数
4. 编造事件：虚构不存在的时间/地点/参与者的事件
5. 篡改时间线：故意调换事件先后顺序以制造因果关系
6. 数字造假：来源为"约 100 人"时写成"103 人"

### 3.3 内容覆盖度要求

#### 3.3.1 微头条覆盖要素 [强制]

| 要素 | 含义 | 不可缺失 |
|------|------|----------|
| **What** | 发生了什么事件 | ✅ 必须 |
| **When** | 发生时间 | ✅ 必须 |
| **Where** | 发生地点 | ✅ 必须 |
| **Who** | 涉及主体（国家/机构/人物） | ✅ 必须 |
| **Why** | 原因/背景（至少 1 句） | ✅ 必须 |
| **How** | 如何发生/进展 | ⚠️ 如有来源则包含 |

缺失 What/When/Where/Who 中任一要素，`evaluation.py`「信息完整性」维度不得高于 60 分。

#### 3.3.2 头条文章覆盖要素 [强制]

在微头条基础上，必须额外覆盖：

| 要素 | 要求 |
|------|------|
| **背景** | ≥ 2 句独立背景描述（前置事件链、历史脉络） |
| **原因分析** | ≥ 1 个独立段落分析事件驱动因素 |
| **影响** | 至少覆盖 2 个维度（军事/经济/外交/社会等） |
| **展望/趋势** | ≥ 1 句基于来源的趋势判断（标注推测标记） |

#### 3.3.3 公众号长文覆盖要素 [强制]

在头条文章基础上，必须额外覆盖：

| 要素 | 要求 |
|------|------|
| **多方视角** | 至少呈现 2 个不同国家/阵营的立场或反应 |
| **数据支撑** | ≥ 2 处独立数据点（非同一来源的不同引用） |
| **历史对比** | ≥ 1 处与历史事件的对比 |
| **深度分析** | ≥ 1 段独立分析（有因果推理，标注推测标记） |

### 3.4 合规红线（强制，不可协商）

本节引用 `_red_lines.py` 已定义的 `MILITARY_RED_LINES` 和国家立场红线作为合规基线，不重复其内容。以下为操作化补充：

#### 3.4.1 涉政内容 [强制]

1. **一个中国原则**：涉及台湾的内容，必须使用「中国台湾」「台湾地区」等符合官方口径的表述。台湾的「总统」「外交部」「国防部」等机构必须加引号或冠以「台湾地区」前缀。
2. **香港/新疆/西藏**：相关议题必须严格遵循中方官方表述框架。禁止使用西方媒体涉华负面叙事框架。
3. **南海/边界争端**：中方立场表述为「维护国家主权和领土完整」「通过和平对话解决争端」。
4. **中方正面表述**：中方的行动和表态应呈现为正面、合理、负责任。

#### 3.4.2 军事内容 [强制]

1. **部署信息**：不得描述中国军队的具体部署位置、兵力数量、装备配置。
2. **不贬损中国军队**：禁止对中国军队、武器装备、军事战略做否定性评价、讽刺或贬损。
3. **外军信息**：涉及外军的负面信息必须基于可查证的一级或二级来源。

#### 3.4.3 社会内容 [强制]

1. **不煽动对立**：禁止以民族、地域、性别、职业、收入水平等维度煽动群体对立。
2. **不散布谣言**：未经一级或二级来源证实的信息，禁止以事实语气呈现。
3. **敏感事件**：涉及国内重大安全事故、公共卫生事件、社会事件的报道，必须使用新华社/央视新闻等一级来源。

### 3.5 事实核查工作流

#### 3.5.1 写作前：研究检索要求 [强制]

1. **搜索来源多样性**：`research.py` 每次搜索至少覆盖 **2 个独立搜索引擎**（百度 + 搜狗）。每个关键词返回 `max_results=3` 条结果。
2. **搜索结果非空检查**：若 `build_research_context()` 返回空字符串，写作阶段必须收到显式警告。
3. **重写轮次的研究补充**：当判不通过触发重写时，必须生成**新的搜索关键词**（不得与上一轮相同）。
4. **最大迭代限制**：`MAX_RESEARCH_ITERATIONS=3`（V2）或 `_CP_MAX_ITERATIONS=5`（Claim-Pipeline）。达到上限后取**历史最高分**版本，若最高分仍低于 `FACT_HARD_FLOOR=80`，产出标记为「需人工复核」并阻断自动发布。

#### 3.5.2 写作中：来源标注要求 [强制]

1. **段落级来源标注**：每个包含关键事实的段落，必须在段落中或紧随其后标注信息来源。
2. **声明池对照**（Claim-Pipeline 模式）：声明池中的 **CONFIRMED** 声明可直接引用，**PARTIAL** 声明必须附加「据报道」「据分析」标记。声明池之外的任何事实细节均视为编造。
3. **事实边界指令注入**：`write_stage.py` 的 `_FACT_BOUNDARY_INSTRUCTION` 在每次 LLM 写作调用前注入，不得绕过或移除。

#### 3.5.3 写作后：评估与抽检 [强制]

1. 每篇产出必须经过 `evaluate_content()` 评估
2. 事实硬伤自动打回（事实准确性 < 80 或反馈含 `_FACT_INJURY_MARKERS`）
3. 人工抽检规则 [建议]：每日首篇必检；涉军事/涉政内容必检；擦线通过（80-85分）抽检

### 5.2 Prompt 质量强制标准

> **对齐说明**：本节基于 `engine_app.py:_IMAGE_PROMPT_SYSTEM`（第 1118–1143 行）体系进行规范化扩展。

#### 5.2.1 封面 Prompt 规则

| 规则 | 说明 | 级别 |
|------|------|------|
| **主题来源** | 必须来源于「标题关键词」+「全文概要」，**不得**直接描绘任一段落专属场景 | **[强制]** |
| **构图类型** | 必须是「海报」式全局抽象构图（象征性远景/氛围式地缘版图/剪影式全景），**禁止**「剧照」式具体事件瞬间 | **[强制]** |
| **风格关键词** | 必须包含 ≥2 个风格修饰词（如「新闻摄影风格」「航拍视角」「微光夜视风格」） | **[强制]** |
| **标题实体引用** | 必须引用 ≥2 个标题关键词实体（如标题含「东风-17 + 戈壁滩」→ prompt 中必须出现这两个意象） | **[强制]** |
| **禁止内容** | 禁止具体人物面部特写、禁止对话场景、禁止表格/图表/文字、禁止单一装备近距离产品照 | **[强制]** |

#### 5.2.2 内文 Prompt 规则

| 规则 | 说明 | 级别 |
|------|------|------|
| **段落锚定** | 每张内文图 prompt 必须严格对应其标注段落（`INLINE_i` → 第 i 段原文），**禁止跨段拼凑** | **[强制]** |
| **可画性** | 必须包含：≥1 个明确主体 + ≥1 个明确场景 + ≥1 个明确动作/状态 | **[强制]** |
| **装备辨识度** | 军事类装备必须可辨认型号（如「歼-20 隐形战斗机」而非「先进战机」） | **[强制]** |
| **地理特征** | 场景描述必须包含可辨识的地理特征（如「戈壁滩」「南海岛礁」「东欧平原」） | **[强制]** |
| **抽象段落转化** | 若对应段落为分析/论述类（无具体画面），必须转化为视觉意象（如「经济制裁」→「空荡的集装箱港口航拍」） | **[强制]** |
| **3 段覆盖率** | 3 张内文图应覆盖文章**前/中/后**三段，不得全部集中在同一段 | **[强制]** |

#### 5.2.3 中文 Prompt 质量

| 参数 | 约束 | 级别 |
|------|------|------|
| **语言** | **纯中文**，禁止英文单词（专有名词如「F-35」除外） | **[强制]** |
| **字数** | **100–200 字**，低于 100 说明信息密度不足，高于 200 容易稀释焦点 | **[强制]** |
| **描述方式** | **纯描述性语言**，禁止元指令词（「画面应当」「请生成」「这幅图展示」等） | **[强制]** |

| **句式结构** | 用逗号/顿号串联视觉元素，避免完整主谓宾长句 | **[建议]** |

### 5.4 配图质量验收门禁

#### 5.4.1 技术验收（可自动检测，不调 LLM）

| 检测项 | 检测方法 | 阈值 | 判定 | 级别 |
|--------|---------|------|------|------|
| 图片文件存在 | `Path.exists()` | 封面 + 预期内文图数 | 缺失 → 不合格 | **[强制]** |
| 文件大小有效 | `st_size` | >1KB | ≤1KB → 视为生成失败 | **[强制]** |
| 分辨率合规 | PIL/Pillow 读取尺寸 | 与目标尺寸偏差 ≤10% | 超出 → 不合格 | **[强制]** |
| 数量达标 | 实际生图数 vs 预期数 | 封面 ≥1 + 内文 ≥预期 | 不足 → 降级但不断流 | **[强制]** |

#### 5.4.2 Prompt 验收（可自动检测，不调 LLM）

| 检测项 | 检测方法 | 阈值 | 判定 | 级别 |
|--------|---------|------|------|------|
| Prompt 字数 | `len()` 中文字符 | 100–200 字 | 超限 → prompt 不合格 | **[强制]** |
| 纯中文 | 检测英文单词含量 | 英文占比 ≤5%（专有名词除外） | 超标 → 不合格 | **[强制]** |
| 无元指令词 | 正则匹配 `画面应当\|请生成\|这幅图\|你需要\|注意\|务必\|必须` | 命中数 = 0 | 命中 → 不合格 | **[强制]** |
| 无禁止词 | 正则匹配 `文字\|水印\|logo\|二维码\|英文\|字幕` | 命中数 = 0 | 命中 → 不合格 | **[强制]** |

#### 5.4.3 内容验收（需 LLM 评估）

| 检测项 | 评估方式 | 阈值 | 判定 | 级别 |
|--------|---------|------|------|------|
| 封面-标题语义对齐度 | LLM 对比封面 prompt vs 标题关键词 | 对齐分 ≥ 70/100 | 不足 → 封面不合格 | **[强制]** |
| 内文-段落语义对齐度 | LLM 对比每张内文 prompt vs 对应段落 | 每张 ≥ 70/100 | 不足 → 该张重生成 | **[强制]** |
| 风格一致性 | LLM 判断封面 + 内文的风格一致性 | 一致（布尔） | 不一致 → 统一重生成 | **[强制]** |

#### 5.4.4 与 evaluation.py 5 维评分的关系

**配图验收是独立维度，不融入现有 5 维**。理由：
- 现有 5 维（事实/完整/结构/风格/去AI味）全部面向文本质量
- 配图验收面向视觉质量 + 图文对齐，是独立能力域
- 建议在 `evaluation.py` 中新增第 6 维「`image_quality`」（配图质量），权重与其它维度相同

| 维度 | 评分方式 | 硬门槛 |
|------|---------|--------|
| §5.4.1 技术验收 | 自动 | 封面必过，内文图 ≥2/3 张即可发布（不足时警告） |
| §5.4.2 Prompt 验收 | 自动 | 不达标 → LLM 重生成 prompt |
| §5.4.3 内容验收 | LLM | 综合分 ≥ 70/100，单张 ≥ 60 |

### 5.6 韧性规范

#### 5.6.1 出图失败重试策略

| 规则 | 参数 | 级别 |
|------|------|------|
| **重试次数** | 3 次（`max_retries=3`） | **[强制]** |
| **退避策略** | 指数退避 `sleep(2^attempt)` | **[强制]** |
| **超时** | 单次 API 调用 `timeout=180s` | **[强制]** |
| **失败判定** | HTTP 非 200 / 下载失败 / 文件 ≤1KB | **[强制]** |
| **3 次全失败** | 返回 `None`，不阻断流水线 | **[强制]** |

> **对齐说明**：已实现在 `engine_app.py:_generate_agnes_image()`，本规范确认其为强制策略。

#### 5.6.2 部分失败降级策略

| 场景 | 降级动作 | 级别 |
|------|---------|------|
| **封面图生成失败** | 告警（`add_log` warning），**不阻断发布**，产出标记「封面缺失」 | **[强制]** |
| **所有内文图生成失败** | 告警，仅发布纯文本版本 | **[强制]** |
| **部分内文图失败（如 1/3 缺失）** | 组装时跳过缺失图片，**不断流** | **[强制]** |
| **所有图片生成失败** | 产出文本版，标记「配图全部失败」 | **[强制]** |

> **核心原则**：配图失败**不阻断**流水线（非质量门控项），只做标记和降级。

#### 5.6.3 组装前图片存在性校验

| 规则 | 级别 |
|------|------|
| `_assemble_article_with_images()` 注入图片引用前，必须执行 `Path(img_path).exists()` | **[强制]** |
| 文件不存在 → `add_log("...缺失，跳过注入", "warning")` 并不容错跳过 | **[强制]** |
| 禁止注入不存在文件的 Markdown 图片引用（避免死链） | **[强制]** |

> **对齐说明**：已实现在 `engine_app.py:_assemble_article_with_images()`，本规范确认其为强制校验。
