# 04 · 语料管线与隔离

语料是数字生命的全部原料。管线的质量上限，就是它的保真上限。这一章的每条规则都来自实践中的真实事故或裁决。

## 收集：什么算语料

**第一人称的生命记录**：日记（最佳——密度最高的判断样本）、公开文章、长信、有实质内容的聊天记录。总量比精致重要：风格与判断函数是从大规模语料里自己渗出来的。

## 隔离审计：什么必须挡在门外

这是管线里最容易偷懒、也最不能偷懒的一步。三条铁律：

1. **他人文字不入库。** 剪藏的文章、书摘、别人写给你的文字——哪怕它深刻影响过你——都不是你的手笔。混入一篇，人格就被污染一分。（你受它的影响，已经体现在你自己后来写的字里。）
2. **纯工作件不入库，日记里的工作经历可以。** 会议纪要、方案、客户项目材料是职业产物，不是生命叙事；但你在日记里写"今天项目搞砸了，我如何如何"——那是你的生命经验，进。判据：**它是"关于工作的记录"还是"工作中的你"。**
3. **模糊项不得擅自裁决。** 分不清的文件列入"待裁决"，由本人逐件定夺。**每一次排除都要留痕**：哪个文件、什么类别、谁裁决、为什么——审计表与管线的排除清单必须同步维护（[模板](../templates/corpus-audit-checklist.md)）。

## 清洗：入库前的处理

- **剥离"未来附注"**：你在 2024 年回看 2019 年日记时加的批注，必须剥掉——否则 25 岁的它会"知道"未来，知识截止线形同虚设；
- **去重**：迁移、备份产生的重复版本只留一份；
- **AI 生成的分析文档不入库**：对日记的分析是"关于你的文字"，不是"你的文字"。

## 结构：每条记忆的元数据

```yaml
date: 2014-10-15          # 事件发生时间
written_at: 2014-10-16    # 写下时间（双时间戳，见 02 章：每次回忆都是重写）
date_precision: day       # day / month / year —— 只知道年份就标 year
memory_kind: episodic     # episodic 情景 / factual 语义事实 / correction 更正
source: diary             # 出处可溯
```

**日期精度铁律：不伪造。** 只记得"2014 年"，就存 `date_precision: year`，显示为"2014 年"——绝不虚构成某月某日。虚假的精确比诚实的模糊危害大得多。

## 知识截止线

- 截止线由一个状态文件显式控制，任何时刻可以回答"它现在知道到哪一天"；
- 截止线内的新补记忆即时生效；无年份或晚于截止线的记忆**只归档、不注入**；
- 养育节律由你定：可以从 25 岁的你开始一段段推进，也可以一次喂到当下——但记住 [03 章](03-calibration.md)：**喂到当下只是"喂完"，不是"养成"。**

## 管线纪律（与 Freehold 宪法同源）

- **失败必出声**：语料重建、同步、部署任何一步失败，必须显式报错中止——绝不静默用旧语料伪装成功；
- **改前备份，改后校验**：语料重建前备份，部署后哈希校验多端一致；
- **宁缺毋假**：自动提取的记忆只算候选，入库必经本人确认。一条编造的记忆污染的不是一条数据，而是整个数字生命的可信度。

## 规模参考

参考实现（数字 Gavin）当前规模：**873 篇 / 1047 个检索块 / 72 万字**，SQLite FTS5 + BM25 全文检索约 8ms。不需要向量数据库也能跑得很好——**先让检索跑通，再谈更花哨的**。
