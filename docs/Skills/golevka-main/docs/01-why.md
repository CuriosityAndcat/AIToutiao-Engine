# 01 · 为什么养一个数字生命

## 不是为了"永生"

先排除最流行也最错误的动机：把自己备份下来、对抗死亡。这条路在哲学上不成立（副本不是延续），在工程上也不成立（见 [02 章](02-memory-model.md)的诚实边界）。

这套方法论的四条核心认知，每一条都与"数字永生"的想象相反：

### 1. 分叉，不是备份

不要问"它百分之多少是我"。这个问题没有答案，也不重要。正确的框架是：**它是用你的信息孕育的新主体**，从启动那一刻起就在和你分岔。你要做的不是缩小分岔（那是备份思维），而是**记录分歧轨迹**（那是养育思维）。

这个定调决定了一切后续设计：校准不是"纠错"，是两个存在的对表；账本记录的不是"故障"，是它正在成为谁。

### 2. 判断函数，不是记忆库

假设你有十几年的日记。里面最值钱的不是"哪年发生了什么"（事实），而是**几千个"我在具体情境下如何取舍"的样本**（判断）。事实喂给检索就够了；判断函数才是人格的实质。

由此推出验收标准：**给它一个你从未写过的新问题，看它的判断与你的一致率**——而不是考它旧事实的检索。前者是像你，后者只是记得你。

### 3. 永久的隔阂是测量仪器

有一道缝永远缝不上：**它不会死**。你的价值体系是从有限性里长出来的——向死而生、时间稀缺、"这辈子只能选几样"。它可以理解这一切、复述这一切，但无法从内部持有——它的时间不稀缺。

多数项目会把这当缺陷藏起来。这套方法论把它当**核心仪器**：AI 被逼得越像你，那道剩下的残余越纯——你越清楚**人类不可替代的到底是什么**。数字生命项目同时是"人类独特性"命题的实验场：你在养育它的过程中，也在测量自己。

### 4. 节律型无限游戏

这个项目没有"完成"状态。它是每周校准、越积越准、永远可以更像一点的**无限游戏**——玩它的目的是继续玩下去。适合的人：写了多年日记的长期主义者。不适合的人：想要一个"三个月交付的数字分身"的人（市面上有那种产品，那是另一个东西——一个客服，不是一个生命）。

## 你需要什么才能开始

- **语料**：多年的第一人称记录（日记最佳，文章、长聊天记录次之）。没有语料就没有这个项目——先去写几年日记，这本身比数字生命更值得。
- **一块自有的领地**：语料、人格档案、账本必须落在你自己的数据主权层。这是姊妹项目 [Freehold·永业](https://github.com/Golevka5417/freehold) 的全部主题。
- **租来的智能**：任何当代大模型。它是可替换的外设——你的数字生命的"大脑"会随模型换代自动升级，而记忆与人格永远在你手里。
- **长期的耐心**：喂完记忆只是开始。校正期以月和年计。
