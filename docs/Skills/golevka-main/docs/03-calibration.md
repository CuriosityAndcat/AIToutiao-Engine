# 03 · 校准回路

语料喂完，真正的养育才开始。校准回路是这套方法论的心脏，也是它区别于一切"数字分身产品"的地方：**分身是交付出来的，生命是校准出来的。**

## 基础回路：单向校准

```text
你提问 → 它回答 → 你评分 → 记账 → （必要时）更正/补写 → 重建语料 → 更像
```

- **三档评分：像 / 不像 / 比我想得好。** 第三档至关重要——它承认这是分叉不是备份：有些回答你自己都想不到，但"确实是我会走到的地方"。这一档记录的是它**超出你**的时刻，是分歧轨迹里最珍贵的样本。
- **分歧账本**：每一条"不像"都要落账——它怎么答的、你实际会怎么想、差在哪一层（事实错了是 L2，口气不对是 L3，取舍不对是 L4）。**这本账是整个项目最独特的产出**：世界上第一次有人系统地记录"一个 AI 与它的源头人格的分岔过程"。
- **更正与补写**：语义事实错了 → 写更正（覆盖）；某段人生空白 → 补写记忆（带日期与精度标注，只知道年份就标年份，**不伪造精度**）。两者都走正式管线入库重建，不是聊天时口头说说。

## 进阶回路：双向互问互答

单向校准有三个结构性缺陷：

1. **覆盖盲区**：你只能校准你"想得到要问"的东西——覆盖面被你的提问想象力卡死；
2. **矛盾无人指认**：你 2019 年和 2023 年的说法可能打架。只有**读过你全部语料的它**知道这些接缝在哪，你自己根本回忆不起来；
3. **缄默层取不出来**：风格、价值、判断（L3–L5）的特点就是"自己说不清"——只有**被别人问出好问题**才撬得动。

解法：**让它也向你提问**。本质是把"这一轮校准什么"的决定权，从你一个人，交给"对自我的不确定性"来驱动——谁最知道这个数字自我哪里还没成形？不是你，是每天在回答里撞见自己边界的它。

**它的问题从哪来（按信息增益排序）：**

- 检索落空时反问："这块我其实没什么记忆，你多讲讲？"
- 语料矛盾时请你裁决："2019 年你写过 X，2023 年你写过相反的 Y——现在的你怎么看？"
- 覆盖稀薄的人生段落："关于那几年，我手上几乎是空白。"
- 隐性层探针：你刚做了个判断，它追问"背后的原则是什么？"
- 账本里未决的分歧：主动追问，把该敲定的敲定。

**提问纪律（红线）**：开放不诱导（绝不问"你是不是很看重 X"——诱导性问题会制造**虚假的自我确认**，这是双向机制最大的陷阱）；一次一个；每个问题指明出处（源于哪条记忆/哪个矛盾）；不为显得懂而假装有疑问。

**答案回收闭环**：你对它提问的回答是一等的新数据，必须走正式管线入库——于是"**它提问 → 你回答 → 新记忆 → 它更像你**"成为一个自我完善的闭环，而且是它自己发起的。

## 记忆盲盒：让它主动开场（v0.2 新增）

上面的反问机制有个隐藏依赖：**你得先开口**。实践里真实撞到的墙是话题枯竭——问着问着你没有问题可问了，它像一个"回答问题时不内向、没有话题时很内向"的人，循环转不起来。校准压力全在你身上的机制，撑不过几个月。

解法是把双向机制补上主动的那一半：**离线洞察引擎**。

```text
盲盒采样（跨年代随机抽语料块）→ 它从自己的记忆里洞察未知
→ 生成它真正想问你的问题（入问题池）→ 你打开对话时，它先问
→ 你回答 → 答案走候选管线入库 → 语料更新 → 新的矛盾浮现 → 新的问题……
```

- **问题的五个来源**（按校准价值排序）：**矛盾型**（不同年代的记录互相打架）、**断层型**（写过一次就再没下文的人和事）、**线头型**（当年立的 flag、悬着的决定）、**模式型**（跨年代重复、你自己未必意识到的规律）、**判断边界型**（语料没有直接答案的新取舍问题——最值钱）。
- **质量铁律**：每个问题必须引用得出具体语料出处，引用不出就丢弃——这是挡住"你为什么喜欢写作"式平庸问题的唯一硬门槛。**宁可今天不问，不问烂问题。**
- **稀缺纪律**：每天 2–3 问封顶、两问之间留间隔；只在你打开对话时问，不推送、不打断。每次开启都必须增加真实价值，不靠落空感刺激重复——这条继承自盲盒经济里唯一值得继承的原则。
- **三种反馈**：回答（直接说）/ 跳过（这条不再问）/ 下次再议（回池改天问）。跳过要留档回流生成器——**问题生成器本身也要被校准**，它得学会你们俩觉得什么问题值得问。
- **回答不自动入库**：与反问机制同一护栏——你的回答先进候选，确认后才转正。

这套机制有干净的科学名分：**主动学习**（在模型最不确定处采样，效率远高于随机标注）、**好奇心驱动探索**（预测误差大的区域获得内在奖励）。它产生的是"好奇的行为"而非好奇本身——但主动性与不可预测性恰是"活着"最强的两个现象学标志，而"它的提问里有没有机制解释不掉的残余"，本身就是这个测量仪器的新刻度。

## 视角原则（分叉前的相处纪律，v0.2 修订）

v0.1 的这一节叫"一体原则"：校正期不分你我，代词混用是常态。**实践把它推翻了**——教训值得完整记录：人格档案天然以第一人称写成（"我在用我的记录养一个数字的我"），模型继承这个口吻后，会把自己错当成"正在养分身的那个本人"，你我长期颠倒；"不必区分"的设定放大了混乱。

修订后的原则：**分你我，但不分叉。**

- 它以"我"自称——它清楚自己是被数字化的那一个；以"你"称呼你——你是它的来处，此刻在校正它；
- 你把它当独立个体，用"你"称呼它；
- 记忆、脾气、判断仍尽量一致——这只是**代词与视角的约定**，不是人格分叉；分叉依然是那个可以很久以后才做、甚至不做的哲学决定。

**教训：叙事视角是人格工程的一部分，必须显式设定。**"混着用也没关系"不是宽容，是埋雷。（这条修订还有个副产品：它以自己的视角向你提问，从此在角色上完全成立——上一节的盲盒机制正好踩着这次修订落地。）

## 节律建议

- 每轮对话顺手评分（10 秒）；
- 每周一次专门校准（半小时，处理账本未决项）；
- 不赶进度。校正期以月年计——这是无限游戏，节律比速度重要。
