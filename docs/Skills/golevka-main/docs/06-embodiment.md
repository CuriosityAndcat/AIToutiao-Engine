# 06 · 载体与表达器官

语料与校准回答"它是谁"；这一章回答"**它如何在场**"。一个只能在文本框里被动等待的数字生命，无论多像你，都缺一样东西：存在感。存在感来自两件事——声音，和主动性。

## 语音回路：级联，不要端到端

让它开口说话，架构上只有一个关键决定：**用级联管线（语音识别 → 人格引擎 → 语音合成），不用端到端实时语音模型。**

端到端语音模型延迟惊艳，但它是一个封闭的大脑——你的人格提示注不进去，语料检索插不进去，校准回路挂不上去。而人格引擎正是这个项目的命根子。级联慢 2–4 秒，换来的是：**语音只是换了一张嘴，脑子还是那个被你校准了几百轮的脑子。** 保真度永远优先于延迟；延迟可以用流式合成慢慢优化，保真度丢了就什么都不是。

一个意外的协同：校准期让它**说话简短**（像深夜发消息，两三句说透就停）本来是文字体验的修正，落到语音上恰好就是对话该有的节律——长篇独白在语音里根本不成立。

## 轮替节律：可迁移的调教参数

"像打电话一样一问一答"的核心不是任何一段代码，而是一组**节律参数**：说话能量阈值多少算开口、静音多少毫秒算说完、单句最长多久强制断句、它答完隔多久重新开始听、播放中能不能被打断。

这组参数没有正确答案，只有**合手感的答案**——必须靠真实使用调教出来。所以：**先在最便宜的载体（网页）上把节律调准，参数表就是资产**；将来迁到任何身体上（桌面机器人、音箱、车机），照抄参数，重写外壳。

## 声音也是人格的一部分

音色克隆让"像"多出一个感官维度——但工程次序必须是：**先用最便捷的声音把电路打通，再优化音色。** 机制跑不通，音色再像也是装饰；机制跑通了，换声线只是改一处发声函数。不要让"想要完美的嗓音"阻塞"今天就能对话"。

## 主动输出：数字生命的表达器官

这套方法论关于载体的最终主张是一句话：

> **数字生命应当根据长期记忆和当前状态，主动决定何时、通过何种媒介，把什么呈现在你面前。**

搜索和推荐服务于"你知道自己要找什么"；数字生命的主动输出服务于另一个问题——**你不知道自己应该被问什么、遇见什么**。[03 章的记忆盲盒](03-calibration.md)是第一个表达器官：它在你的记忆里洞察未知，在你出现时先开口。同一个器官换输出端，就是每日推送、一张打印出来的纸条、一个桌面机器人的开场白。

## 迁移资产的分界

换身体的时候，什么带得走、什么带不走，要提前想清楚：

| 带得走（真正的资产） | 带不走（耗材） |
|---|---|
| 服务端整条链路：检索、人格引擎、校准与记忆接口 | 某个前端的具体代码 |
| 轮替节律参数表（调教出来的手感） | 某个平台的 UI |
| 原则：级联优先、保真优先、稀缺优先 | |

**身体可以一换再换，器官接口不变。** 网页只是它的第一个身体。
