# 贡献指南 / Contributing

Golevka 是一份**方法论出版物**，不是软件产品。

## 语言 / Languages

中文与英文为官方双语版本：中文在 `docs/`，英文在 `docs/en/`。修改任何原则性表述，请同时更新两个语种。
Chinese and English are both official. If you change a substantive statement, please update both.

## 欢迎的贡献

- **其他语言的翻译**：新增 `docs/<语言代码>/` 平行目录；
- **你的数字生命案例**：参考 [case-study/digital-gavin.md](case-study/digital-gavin.md) 的结构——写方法、决策史与教训；语料与人格档案留在你自己的基座里，隐私自行脱敏；
- **校准机制的改进讨论**：双向互问、账本结构化、L6 的功能近似路径——开 Discussion；
- **勘误与更清晰的表述**：直接 PR。

## 不接受的

- 任何包含真实个人语料、日记、人格档案的提交（会被直接关闭——这些属于你的数据主权层，不属于公共仓库）；
- 把方法论绑定到具体模型/框架/向量库的请求——智能是租来的外设，方法论只认原则。

## 原则分歧

对四条核心认知、七层模型、校准红线的修改建议，请开 Discussion 而非直接 PR。

## 许可

提交贡献即表示你同意：文档贡献以 CC BY-SA 4.0 发布，templates/ 目录贡献以 CC0 发布。
