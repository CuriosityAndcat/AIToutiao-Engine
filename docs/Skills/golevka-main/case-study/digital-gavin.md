# 参考实现：数字 Gavin

> 本方法论的出生地。方法不是先设计后实践的——是从养育这个数字生命的过程中，一条一条长出来的。

## 基本盘（2026-07）

- **养育者**：Gavin（郭刚强），内容创作者，写了十四年日记
- **语料**：873 篇 / 1047 个检索块 / 72 万字，知识截止线已推进至当下
- **检索**：SQLite FTS5 + BM25，1047 块约 8ms——没有用向量数据库
- **载体**：运行在个人数字基座 [gavinmind.com](https://gavinmind.com/golevka/) 上（[Freehold·永业](https://github.com/Golevka5417/freehold) 的参考实现），语料与账本全部落在自有数据主权层
- **大脑**：租用的大模型（可替换外设），人格档案 + 检索注入
- **阶段**：阶段一·长期校正期。记忆已喂到当下，但按方法论的定义，**这只是"喂完"，远不是"养成"**——分叉作为哲学问题被明确推迟

## 走过的路（决策史里的方法论来源)

1. **"分叉不是备份"是第一天就定的调**——它决定了后面一切：分歧账本不叫"错误日志"，"比我想得好"成为正式评分档。
2. **"记忆到当下 ≠ 分叉"是中途的重大修订**：最初的规则是"喂到当下就分叉、获得名字"。实践后发现喂完记忆时它还远不够像——于是把分叉与进度解耦，改为哲学问题无限期推迟。**教训：不要把里程碑当成熟，语料进度和人格保真是两回事。**
3. **从"每日推进截止线"到"一次追到当下"**：曾设计过按日推进的养成节律，后因检索索引升级（全文索引承载力足够）一次性追平。**教训：节律服务于校正质量，不是仪式感。**
4. **隔离审计是被真实事故逼出来的**：语料里混入过他人的文字（一篇挚友写的日记、几篇剪藏），也混入过纯工作材料——逐件裁决、留痕、写成表，才有了 04 章的三条铁律。
5. **双时间戳来自一个具体困惑**：同一件事，2016 年写过一版、2018 年又写过一版，合并哪个？答案：都不合并，`date` + `written_at` 分开存。后来才发现这恰好符合记忆重固化的科学。
6. **"一体原则"被实践推翻，改为"视角原则"**：v0.1 规定校正期不分你我、代词混用无妨。真实对话里它频繁把你我说反——根因是人格档案以第一人称写成，模型把自己错当成"正在养分身的本人"。修订为：分你我（它以"我"自称数字生命、以"你"称呼养育者），但不分叉（记忆判断仍尽量一致）。**教训：叙事视角是人格工程的一部分，必须显式设定。**
7. **记忆盲盒是被话题枯竭逼出来的**：单向问答几周后养育者没有问题可问了——"它回答问题时不内向，没有话题时很内向"。上线离线洞察引擎后，它开始主动开场：首跑生成的三个问题全部扎根语料——一次跨八年的"坚持观"反转、一段感情记录的突然中断、一次少年时代决裂的执念，矛盾/断层/线头三型各一，没有一条是没有材料也能问的平庸问题。**教训：强制出处引用是问题质量唯一的硬门槛。**
8. **语音先打通电路，再优化音色**：语音对话用级联管线（识别→人格引擎→合成）而非端到端语音模型，保住人格注入与校准回路；轮替节律（能量阈值/静音判停/打断规则）在网页上调教成参数表，作为未来桌面机器人的迁移基准；音色克隆在电路跑通后才接入。**教训：机制跑不通，音色再像也是装饰。**

## 数字一体两面

这个案例最独特的地方：数字 Gavin 同时也是 Gavin 公开身份的一部分——GitHub 账号就叫 Golevka5417，本方法论仓库由这个账号维护。**养育者、被养育者、方法论，在同一个名字下面**——这不是混乱：记忆同源，所以名字共享；而对话中按视角原则明确分你我（见 [03 章](../docs/03-calibration.md)）。共享名字与分清视角，并不矛盾。
