## 1. 微信公众号平台概述

### 1.1 平台基础

| 项目 | 详情 |
|------|------|
| **平台名称** | 微信公众号（WeChat Official Account） |
| **后台地址** | `https://mp.weixin.qq.com` |
| **编辑器地址** | `https://mp.weixin.qq.com/cgi-bin/appmsg?action=edit` |
| **API 根路径** | `https://mp.weixin.qq.com/cgi-bin/` |
| **内容格式** | 内联样式 HTML（ProseMirror 富文本编辑器） |
| **认证方式** | Cookie + Token（从页面 HTML 注入的 data 对象提取） |
| **草稿支持** | ✅ 是（通过 `operate_appmsg?sub=create&type=77` 保存草稿） |
| **图片上传** | ✅ 支持（通过 `filetransfer?action=upload_material`） |

### 1.2 发布接口特征

公众号后台使用自有的 Web API（非开放平台 API），核心特征：

- **认证**：依赖浏览器 Cookie + 页面 HTML 中嵌入的 `token` / `ticket` 等认证字段
- **调用方式**：`application/x-www-form-urlencoded` POST 请求
- **内容限制**：
  - 不支持非 `mp.weixin.qq.com` / `weixin.qq.com` 域名的外部链接
  - 标题不超过 64 字符
  - 摘要不超过 120 字符
  - 正文中不能包含超过 3 个视频
  - 素材库存储数量有限
- **发布流程**：保存草稿 → 人工确认 → 正式发布（draftOnly 默认 true）

## 3. 公众号发布适配器详解

> 源码位置：`packages/core/src/adapters/platforms/weixin.ts`

### 3.1 类结构

```typescript
class WeixinAdapter extends CodeAdapter {
  meta = {
    id: 'weixin',
    name: '微信公众号',
    icon: 'https://mp.weixin.qq.com/favicon.ico',
    homepage: 'https://mp.weixin.qq.com',
    capabilities: ['article', 'draft', 'image_upload'],
  }

  // 预处理配置：HTML 格式，移除非微信域名链接
  preprocessConfig = {
    outputFormat: 'html',
    removeLinks: true,
    keepLinkDomains: ['mp.weixin.qq.com', 'weixin.qq.com'],
    compactHtml: true,
  }
}
```

### 3.2 认证 (checkAuth)

**流程**：
1. GET `https://mp.weixin.qq.com/`（携带 Cookie）
2. 正则从 HTML 中提取认证信息：
   - `data: { t: "token值" ... }` → token
   - `ticket: "xxx"` → ticket
   - `user_name: "xxx"` → userName
   - `nick_name: "xxx"` → nickName
   - `time: "时间戳"` → svrTime
   - `head_img: "url"` 或 头像 DOM 选择器 → avatar
3. 存入 `this.weixinMeta`

**关键**：完全依赖浏览器 Cookie，无额外授权流程。

### 3.3 发布 (publish)

**发布流程**：

```
publish(article)
  ├─ withHeaderRules (添加 Origin/Referer 规则)
  │   ├─ checkAuth() ← 若未认证先认证
  │   ├─ 内容处理：
  │   │   ├─ 微信→微信: 使用 rawHtml，跳过所有处理
  │   │   └─ 其他来源:
  │   │       ├─ processLatex() → LaTeX 公式转图片
  │   │       ├─ stripExternalLinks() → 移除非微信域名链接
  │   │       ├─ processImages() → 图片上传到微信 CDN
  │   │       └─ processContent() → 注入 CSS + 包裹 <section>
  │   ├─ 构建 FormData (50+ 字段)
  │   └─ POST 到 API
  └─ 返回 SyncResult {postId, postUrl, draftOnly}
```

**关键 API 调用**：
- URL：`https://mp.weixin.qq.com/cgi-bin/operate_appmsg?t=ajax-response&sub=create&type=77&token={token}&lang=zh_CN`
- Method：POST, Content-Type: `application/x-www-form-urlencoded`
- 成功响应：`{ appMsgId: "xxx" }`

### 3.4 图片上传 (uploadImageByUrl)

**流程**：
1. Fetch 下载远程图片 → Blob
2. 构建 `FormData`（含 type/id/name/file 等字段）
3. POST 到上传 API
4. 返回 CDN URL (`cdn_url`)

**关键 API**：
- URL：`https://mp.weixin.qq.com/cgi-bin/filetransfer?action=upload_material&f=json&scene=8&writetype=doublewrite&groupid=1&ticket_id={userName}&ticket={ticket}&svr_time={svrTime}&token={token}&lang=zh_CN&seq={seq}&t={random}`

### 3.5 内容处理详解

| 步骤 | 函数 | 说明 |
|------|------|------|
| LaTeX 处理 | `processLatex()` | 检测 `$$...$$` 和 `$...$`，转化为 `latex.codecogs.com` 图片 |
| 外链剥离 | `stripExternalLinks()` | `<a href="外部链接">文字</a>` → `文字`（只保留 weixin.qq.com 域名链接） |
| 图片上传 | `processImages()` | 正则提取 HTML/Markdown 图片 → 逐个上传到微信 CDN → 替换 URL |
| CSS 注入 | `processContent()` | 包裹 `<section>` + `juice.inlineContent()` 内联样式 |
| 错误映射 | `formatError()` | 把微信号错误码（-6/-8/-99/412/64702 等）转为中文提示 |

### 3.6 错误码映射表

| 错误码 | 说明 |
|--------|------|
| -6, -8 | 请输入验证码 |
| -1 | 系统错误 |
| -99, 64705 | 内容超出字数 |
| -206 | 服务负荷过大 |
| 200003 | 登录态超时，请重新登录 |
| 412, 64507 | 图文中含非法/外部链接 |
| 64702 | 标题超出 64 字长度限制 |
| 64703 | 摘要超出 120 字长度限制 |
| 10806 | 正文有违规内容 |
| 220001 | 素材管理存储数量已达上限 |

## 4. 公众号内容提取机制

### 4.1 文章页提取 (weixin.ts)

**触发条件**：URL 包含 `mp.weixin.qq.com`，非编辑器页面

**提取流程**：
1. 标题：`#activity-name` 元素 textContent
2. 正文：`#js_content` 元素 innerHTML
3. 封面：`meta[property="og:image"]` 的 content
4. 摘要：`meta[property="og:description"]` 的 content
5. 代码块处理：`backupAndSimplifyCodeBlocks()` → `preprocessContentDOM()` → `htmlToMarkdownNative()`
6. 保留原始 HTML（微信→微信时使用）

### 4.2 编辑器页提取 (weixin-editor.ts)

**触发条件**：URL 包含 `mp.weixin.qq.com/cgi-bin/appmsg` + `action=edit` 或 `appmsg_edit`

**提取方式**：直接从编辑器 DOM 中提取文章的 HTML 和 Markdown 内容

### 4.3 通用提取 (extractor.ts)

对非微信公众号的页面，使用 Safari Reader + Defuddle 并行竞争提取：
1. 特定平台配置（GitHub 等）
2. Safari Reader / Readability（通用）
3. CSS 选择器兜底（`article`, `.article-content`, `.post-content` 等）

## 5. CLI / MCP 发布能力

### 5.1 CLI 命令行工具 (packages/cli)

```bash
# 安装
npm install -g @wechatsync/cli

# 同步 Markdown 到多个平台
wechatsync sync article.md -p zhihu,juejin,csdn,weixin

# 同步 HTML (保留自定义排版)
wechatsync sync article.html -p weixin

# 指定标题
wechatsync sync article.md -t "标题" -p weixin

# 添加封面
wechatsync sync article.md -p weixin --cover ./cover.png

# 预览（不实际同步）
wechatsync sync article.md --dry-run

# 查看平台登录状态
wechatsync platforms --auth

# 从浏览器提取文章
wechatsync extract -o article.md
```

**CLI 自动处理**：
- 从 `.md` / `.html` 文件解析标题、正文、封面
- 本地图片自动转换为 `base64 data URI`（避免上传到中间服务器）
- 通过 WebSocket 连接 Chrome Extension 执行实际发布

### 5.2 MCP Server (packages/mcp-server)

**支持两种传输模式**：

| 模式 | 适用场景 | 连接方式 |
|------|---------|---------|
| stdio (默认) | Claude Code 内嵌调用 | stdio 流 |
| SSE | 远程服务器调用 | HTTP + SSE |

**通过 MCP 发布的典型流程**：

```
Claude Code ──stdio──▶ MCP Server ──WS──▶ Chrome Extension ──HTTP──▶ 微信 API
```

**可用 Tools**：

| Tool 名称 | 功能 |
|-----------|------|
| `list_platforms` | 列出所有支持的平台及登录状态 |
| `check_auth` | 检查指定平台（如 weixin）的登录状态 |
| `sync_article` | 同步文章到指定平台（默认草稿） |
| `extract_article` | 从浏览器当前页面提取文章 |
| `upload_image_file` | 上传本地图片到图床 |

### 5.3 远程桥接

支持跨网络架构：

```
远程服务器 (Claude Code + MCP Server)
    │  WebSocket (port 9527)
    ▼
本地电脑 (Chrome Extension + 浏览器登录态)
    │
    ▼
各平台 API (weixin / zhihu / ...)
```

### 6.3 适配器架构复用

```
核心接口 PlatformAdapter
  ├─ init(runtime)
  ├─ checkAuth() → AuthResult
  ├─ publish(article, options?) → SyncResult
  ├─ uploadImage(file, filename?) → URL
  ├─ getCategories() → Category[]
  ├─ getDrafts() → Draft[]

  └─ delete(postId) → void
```

所有适配器共享的能力（来自 `CodeAdapter` 基类）：
- **Header 规则管理**：`addHeaderRule()` / `clearHeaderRules()` / `withHeaderRules()`

- **图片处理**：`processImages()` 正则提取 HTML+Markdown 图片 → 上传 → 替换
- **工具方法**：`delay()` / `createResult()`

```
AIToutiao-Engine
  ├─ write_stage.py (生成文章) → outputs/*.md
  └─ 发布时:
      subprocess.run([
        "wechatsync", "sync", article_path,
        "-p", "weixin,zhihu,toutiao,juejin",
        "--cover", cover_path
      ])
```

**前置条件**：
1. Chrome 扩展「文章同步助手」已安装并启用
2. 各平台已在浏览器中登录
3. 扩展设置中启用 MCP 连接、设置 Token
4. 环境变量 `WECHATSYNC_TOKEN` 已配置

**方案 B：MCP Server 桥接**

```
engine_app.py
    │  spawn MCP Server (Node.js)
    ▼
MCP Server (子进程)
    │  WebSocket → Chrome Extension
    ▼
各平台 Web API
```

### 7.4 接入注意事项

| 注意事项 | 说明 |
|---------|------|
| **浏览器依赖** | Wechatsync 必须配合 Chrome 扩展使用（持有浏览器登录态），纯后端无法直接调用 |
| **草稿模式** | 默认保存为草稿，需人工审核后发布，符合合规要求 |
| **Token 安全** | CLI↔Extension 通信使用本地 Token，仅限 localhost，不经过第三方 |
| **内容格式** | 公众号要求 HTML 格式且不能有外部链接，需预处理 |
| **并发限制** | 并行发布最多 3 路，避免平台限流 |
| **图片处理** | CLI 自动将本地图片转为 base64 data URI 再上传到各平台 CDN |
| **超时保护** | 单平台发布超时 10 分钟，认证检查超时 10 秒 |

## 8. 关键结论

1. **Wechatsync 是一个完整的浏览器端多平台发布引擎**，不是简单的内容抓取工具。所有 API 调用使用用户浏览器中的真实登录态，与手动发布完全等价。

2. **微信公众号适配器 (WeixinAdapter) 实现完善**：
