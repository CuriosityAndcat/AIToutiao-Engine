## 第 1 步：注册微信公众号

### 1.1 确定账号类型

| 类型 | 适用 | 区别 |
|------|------|------|
| **订阅号** | 个人/媒体，每日可群发 1 条 | ✅ 推荐，个人可注册 |
| **服务号** | 企业/组织，每月 4 条 | 需要营业执照，不适合 |
| **企业微信** | 企业内部管理 | 不适合内容发布 |

> **如果你是个人运营 → 选「订阅号」**。我们项目的军事号定位也是订阅号模式。

### 1.2 注册步骤

1. 打开 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击「立即注册」
3. 选择账号类型 → **订阅号**
4. 填写邮箱（一个邮箱只能注册一个公众号）
5. 查收邮箱验证码并验证
6. 选择注册地 → **中国大陆**
7. 信息登记：
   - **个人**：上传身份证正反面 + 手机号验证
   - **审核时间**：1-7 个工作日（通常 1-2 天）
8. 设置账号名称、功能介绍、头像
   - **名称**：建议包含领域关键词（如「XX 军事观察」「XX 美食记」）
   - **功能介绍**：一句话说清账号定位
   - **头像**：清晰简洁 logo，建议 200x200 以上

### 1.3 注册后必做配置

```text
☐ 登录后台 → 设置 → 公众号设置 → 完善账号信息
☐ 设置 → 安全中心 → 绑定运营者微信号（至少 2 个，防丢失）
☐ 设置 → 隐私 → 确认是否开启「允许搜索」
☐ 功能 → 自动回复 → 设置关注后自动回复（引导语）
☐ 功能 → 自定义菜单 → 设计 3 个主菜单
```

## 第 2 步：安装 Wechatsync Chrome 扩展

> Wechatsync 是连接 AIToutiao-Engine 和微信公众号的桥梁。
> 它是一个 Chrome 浏览器扩展，使用你浏览器中的登录态来发布文章，不经过任何第三方服务器。

### 2.1 安装扩展

**推荐方式**：从 Chrome 网上应用店安装（自动更新）

1. 打开 Chrome 浏览器
2. 访问 [Chrome 网上应用店 - 文章同步助手](https://chrome.google.com/webstore/detail/%E5%BE%AE%E4%BF%A1%E5%90%8C%E6%AD%A5%E5%8A%A9%E6%89%8B/hchobocdmclopcbnibdnoafilagadion)
3. 点击「添加至 Chrome」
4. 确认权限（需要 cookies、storage、declarativeNetRequest etc.）

**备用方式**（手动安装）：
1. 下载 [最新 Release ZIP](https://wpics.oss-cn-shanghai.aliyuncs.com/wechatsync-2.0.9.zip?date=20260324)
2. 解压到本地目录
3. Chrome 地址栏输入 `chrome://extensions/`
4. 打开右上角「开发者模式」
5. 点击「加载已解压的扩展程序」
6. 选择解压后的目录

### 2.2 验证安装

- Chrome 工具栏出现绿白色「同」字图标 ✓
- 点击图标 → 弹出同步面板 ✓

## 第 3 步：登录公众号后台

### 3.1 首次登录

1. 在 Chrome 中打开 [https://mp.weixin.qq.com](https://mp.weixin.qq.com/)
2. 使用注册时绑定的微信号扫码登录
3. 确认进入后台管理页面（能看到数据概览、图文素材等）

### 3.2 验证扩展能识别登录态

1. 保持公众号后台页面打开
2. 点击 Chrome 工具栏的「同」扩展图标
3. 在弹出的面板中，应该能看到「微信公众号」显示为已登录状态
4. 如果没有显示，点击「刷新登录状态」

> ⚠️ **关键**：扩展依赖浏览器中的 Cookie 来识别登录态。
> 如果登出或 Cookie 过期，扩展会提示未登录，重新扫码即可。

### 3.3 日常保持登录

- 公众号登录态通常保持 7-30 天
- 扩展使用 `storage.local` 缓存认证信息，打开面板时优先使用缓存
- 如遇发布失败提示「登录态超时」，手动打开 [mp.weixin.qq.com](https://mp.weixin.qq.com/) 扫码刷新

## 第 4 步：配置 Wechatsync 与 CLI 连接

> CLI 命令行工具调用 Wechatsync Chrome 扩展发布文章。
> 我们在 AIToutiao-Engine 的发布阶段会调用 CLI 来完成多平台发布。

### 4.1 安装 CLI

```bash
npm install -g @wechatsync/cli
```

验证安装：

```bash
wechatsync --help
# 应该输出帮助信息
```

### 4.2 启用 MCP 连接

1. 右键 Chrome 工具栏「同」扩展图标 → 选择「选项」或「设置」
2. 找到「MCP 连接」开关 → 打开
3. 设置一个 **安全 Token**（自己记住，后面会用）
4. 点击「保存」

### 4.3 配置环境变量

在 Windows 中（永久设置）：

```powershell
# 右键"此电脑"→ 属性 → 高级系统设置 → 环境变量
# 新增用户变量：
#   变量名: WECHATSYNC_TOKEN
#   变量值: (你在第 4.2 步设置的 Token)
```

在 PowerShell 中临时设置（每次新窗口都需要）：

```powershell
$env:WECHATSYNC_TOKEN = "你的token"
```

### 4.4 测试连通性

打开一个终端，执行：

```bash
# 检查 CLI 能否连接 Chrome 扩展
wechatsync platforms --auth
```

**预期结果**：
- CLI 会启动 WebSocket 服务（端口 9527）
- 等待 Chrome 扩展连接
- 连接成功后，显示当前已登录的平台列表
- 你应该能看到 **微信公众号: 已登录**

如果长时间卡在「等待 Chrome Extension 连接...」：
1. 确认扩展已安装且 MCP 连接已启用
2. 确认 Token 一致
3. 保持公众号后台页面在浏览器中打开

### 6.3 自动发布函数（供 engine_app.py 调用）

在 `publisher_service.py` 中新增一个函数（或新建 `wechat_publisher.py`）：

```python
import subprocess
import os
from pathlib import Path
from typing import List

def publish_via_wechatsync(
    markdown_path: str,
    platforms: List[str] = ["weixin"],
    cover_path: str = None,
    token: str = None,
    timeout: int = 120,
) -> dict:
    """
    通过 Wechatsync CLI 发布文章到指定平台

    Args:
        markdown_path: Markdown 文件路径（S5 产出）
        platforms: 目标平台 ID 列表，默认 ["weixin"]
        cover_path: 封面图路径（可选）
        token: WECHATSYNC_TOKEN（可选，不传从环境变量读取）
        timeout: 等待扩展连接超时（秒）

    Returns:
        {"success": bool, "results": list, "error": str}
    """
    file_path = Path(markdown_path)
    if not file_path.exists():
        return {"success": False, "error": f"文件不存在: {markdown_path}"}

    cmd = [
        "wechatsync", "sync", str(file_path),
        "-p", ",".join(platforms),
        "--timeout", str(timeout * 1000),
    ]

    if cover_path:
        cmd.extend(["--cover", cover_path])

    env = os.environ.copy()
    if token:
        env["WECHATSYNC_TOKEN"] = token

    try:
        result = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            timeout=timeout + 30,
            env=env,
        )
        if result.returncode == 0:
            return {
                "success": True,
                "stdout": result.stdout,
                "stderr": result.stderr,
            }
        else:
            return {
                "success": False,
                "error": result.stderr or result.stdout,
                "stdout": result.stdout,
            }
    except subprocess.TimeoutExpired:
        return {"success": False, "error": "发布超时"}
    except FileNotFoundError:
        return {
            "success": False,
            "error": "wechatsync 未安装，请执行: npm install -g @wechatsync/cli"
        }
```

### 6.4 在 S5 完成后自动触发

在 `engine_app.py` 的 S5 发布阶段，增加 Wechatsync 发布选项：

```python
# engine_app.py 中新增逻辑（示例）

def step_publish_via_wechatsync(run_dir, platforms="weixin"):
    """通过 Wechatsync 发布到多平台"""
    article_path = Path(run_dir) / "完整稿件_配图版.md"

        str(article_path),
        platforms=[p.strip() for p in platforms.split(",")],
    )
    return result
```

## 第 7 步：日常运营流程

### 7.1 每日操作

```text
早晨：
  ☐ 打开 Chrome → 确认公众号后台已登录
  ☐ 确认 Wechatsync 扩展 MCP 连接已启用

流水线运行：
  ☐ 正常跑 S1-S5 流水线生成内容
  ☐ 运行发布脚本：wechatsync sync outputs/latest/完整稿件.md -p weixin

发布后：
  ☐ 登录公众号后台 → 图文素材 → 草稿箱
  ☐ 预览文章 → 扫码在手机确认排版
  ☐ 检查外链（公众号不允许非微信域名链接）
  ☐ 设置封面图、摘要
  ☐ 点击「保存并群发」
  ☐ 扫码确认 → 发布完成
```

### 7.2 公众号内容规范

| 要求 | 说明 |
|------|------|
| **标题** | 不超过 64 字 |
| **摘要** | 不超过 120 字，会话列表限 34 字 |
| **封面图** | 建议 900x383（大图模式），简洁有冲击力 |
| **正文** | 1500-2500 字最佳，每段不超过 4 行 |
| **外部链接** | ❌ 不允许（Wechatsync 会自动剥离） |
| **图片** | 自动上传到微信 CDN（`mmbiz.qpic.cn`） |
| **群发频率** | 订阅号每日 1 条，建议每周 3-5 条 |

### 7.3 内容策略建议

参考 agency-agents-zh-main 微信公众号专家的建议：

- **保持稳定节奏**：每周 3-5 篇，固定时间推送
- **60/30/10 法则**：60% 价值内容 + 30% 互动内容 + 10% 推广
- **打开率目标**：30%+（行业均值 20-25%）
- **每篇要有行动号召**：引导互动（留言/在看/转发）

## 第 8 步：常见问题排查

### 8.1 扩展无法连接 CLI

```
症状: CLI 提示 "等待 Chrome Extension 连接..." 超时
```

**检查清单**：
- ☐ Chrome 浏览器正在运行（不是后台休眠）
- ☐ 扩展已安装且在 Chrome 工具栏可见
- ☐ 扩展设置中「MCP 连接」已开启
- ☐ 扩展设置的 Token 与 `WECHATSYNC_TOKEN` 环境变量一致
- ☐ 没有其他 CLI 进程占用了 9527 端口

**解决端口冲突**：
```powershell
# 查看谁占了 9527 端口
netstat -ano | findstr :9527

# 使用其他端口
$env:SYNC_WS_PORT = "9600"
wechatsync sync article.md -p weixin
```

### 8.2 发布失败：登录态超时

```
症状: 同步失败，错误: "登录态超时，请重新登录"
```

**解决**：
1. 在 Chrome 中打开 [mp.weixin.qq.com](https://mp.weixin.qq.com/)
2. 重新扫码登录
3. 重新执行发布命令

### 8.3 发布失败：外链问题

```
症状: 同步失败，错误: "图文中含非法外链"
```

**解决**：
- Wechatsync 的 `WeixinAdapter` 会自动剥离外部链接
- 但如果检查仍失败，需要手动检查文章中是否包含 `mp.weixin.qq.com` 之外的超链接
- 可添加 `weixin.qq.com` 到白名单的外链不需要处理

### 8.4 发布失败：标题超长

```
症状: 同步失败，错误: "标题超出64字长度限制"
```

**解决**：
- 缩短标题至 64 字以内
- 可以在 S3 写作时添加标题长度约束

### 8.5 图片上传失败

```
症状: 部分图片显示为空白或裂图
```

**解决**：
- 确认图片 URL 可公开访问
- 微信公众号只允许一种图片 CDN 域名，Wechatsync 会自动上传到 `mmbiz.qpic.cn`
- 检查网络连接是否稳定
- 可以通过扩展的 MCP 上传图片到图床测速：`wechatsync extract`

### 8.6 wechatsync 命令找不到

```powershell
# 重新安装
npm install -g @wechatsync/cli

# 确认安装路径在 PATH 中
where wechatsync
```

## 附录

### A. 一键发布脚本（推荐）

创建 `publish.bat`（Windows）：

```batch
@echo off
setlocal

:: 配置区
set WECHATSYNC_TOKEN=你的token
set ARTICLE_PATH=%1
set PLATFORMS=%2

if "%ARTICLE_PATH%"=="" (
    echo 用法: publish.bat ^<稿件路径^> [平台列表]
    echo 示例: publish.bat outputs\latest\完整稿件_配图版.md weixin,toutiao,zhihu
    exit /b 1
)

if "%PLATFORMS%"=="" set PLATFORMS=weixin

echo ========================================
echo  通过 Wechatsync 发布文章
echo ========================================
echo  稿件: %ARTICLE_PATH%
echo  平台: %PLATFORMS%
echo ========================================

wechatsync sync "%ARTICLE_PATH%" -p %PLATFORMS%

if %ERRORLEVEL% EQU 0 (
    echo.
    echo ✓ 发布完成！请登录各平台后台查看草稿。
) else (
    echo.
    echo ✗ 发布失败，请检查错误信息。
)

endlocal
```

### B. 平台 ID 速查

| 平台 | ID | CLI 参数 |
|------|-----|---------|
| 微信公众号 | weixin | `-p weixin` |
| 今日头条 | toutiao | `-p toutiao` |
| 知乎 | zhihu | `-p zhihu` |
| 掘金 | juejin | `-p juejin` |
| CSDN | csdn | `-p csdn` |
| 微博 | weibo | `-p weibo` |
| 小红书 | xiaohongshu | `-p xiaohongshu` |
