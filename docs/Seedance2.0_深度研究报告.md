## 三、核心哲学：导演思维

### 3.1 一句话总结

> **Cinematic 不是决策，动机才是。**
>
> 导演不是从菜单里挑选镜头/灯光/表演，而是先决定「这场戏应该对观众做什么」，然后让每一个技术选择都服务于这个唯一意图，并贯穿整部影片保持一致的作者风格。

### 3.2 一致性法则（引擎第一定律）

> **一个意图，所有乐器奏同一个音符。**

每一场戏的每个技术选择都必须回答：「这个选择是因为意图是 X 吗？」

| 乐器 | 承载什么 | 如何表达意图 |
|------|----------|-------------|
| **景别** | 距离 = 亲密或审判 | 近景对内心；远景对上下文/孤立/尺度 |
| **角度/高度** | 权力与同情 | 仰拍赋予权力；俯拍削弱；平视=平等 |
| **镜头感** | 心理空间 | 长焦压缩孤立；广角打开空间/制造压力 |
| **摄影机运动** | 观众的冲动 | 推镜=逼近认知；拉镜=放弃/揭示；静止=屏息 |
| **灯光设计** | 情感曝光 | 光比、方向、色温决定安全/威胁、温暖/疏离 |
| **调度与走位** | 空间中的关系 | 接近度、层次、谁进出画框、谁转身 |
| **表演** | 此刻的真相 | 一个可读的物理动作，演出潜文本 |
| **声音** | 耳朵的感受 | 密度、静默、一条动机明确的音效 |
| **剪辑与时长** | 呼吸与压力 | 停留增加重量；剪切释放或加速 |

### 3.3 反废片清单（Anti-Slop）

**拒绝这些套路：**
- ❌ 为炫技而选择技术
- ❌ "电影感/史诗/美丽/高质量"作为空泛目标
- ❌ 一个项目内混合矛盾导演风格
- ❌ 表演写成情绪词（"悲伤/愤怒" → 应写成可见动作）
- ❌ 无动机的移动摄影机
- ❌ 无来源的光
- ❌ 每个乐器单独看不错但指向不同方向

**修复方法：回到一句话意图，让每一个选择服务于它。**

## 四、导演引擎（Directing Engine）：9 步决策流程

这是整个技能包的**核心大脑**，定义了 AI 如何像专业导演一样思考。

### Step 1：导演读解（五问驱动）

| 问题 | 内容 |
|------|------|
| **功能** | 这场戏在整体故事中干什么？（引入 / 深化 / 转折 / 收尾） |
| **转折** | 唯一的价值翻转（安全→威胁，希望→绝望，陌生人→盟友……） |
| **视点与共情** | 观众的身体站在哪里？共情是影像选择之前的物理位置 |
| **权力** | 谁拥有权力？权力如何流动？（高度、大小、谁看谁、谁占据空间） |
| **潜文本** | 没说出口的事实是什么？行为与台词的间隙就是导演工作的空间 |

### Step 2：设置或继承项目风格

六种预设导演声音（**不可混合，全篇一致**）：

| 风格 | 摄影机 | 灯光 | 色彩 | 节奏 | 最佳用途 |
|------|--------|------|------|------|---------|
| 观察自然派 | 不可见/动机化/静止或微手持 | 可用/柔 | 真实克制 | 长停留/晚剪 | 接地戏剧/纪录片感 |
| 构图古典派 | 刻意/平衡/测量运动 | 雕塑般/干净 | 受控和谐 | 耐心/在节拍上 | 高端/时代剧/精品广告 |
| 动态直觉派 | 手持能量/跟拍/近 | 硬/高对比 | 高饱和褪色 | 快/打击乐 | 动作/运动/宣传片 |
| 表现风格派 | 设计运动/大胆构图 | 戏剧化 | 饱和/标志性调色 | 音乐性/节奏 | MV/时尚/奇幻 |
| 亲密极简派 | 近镜头/小运动 | 单一柔源 | 暖或克制 | 慢/少剪 | 个人/情感/孤独 |
| 图形形式派 | 锁定/精确/几何 | 硬/塑形 | 有限/刻意 | 精确/平 | 设计导向/冷幽默 |

### Step 3：推导统一设置

基于场景类型 + 一句话意图，选择所有乐器的具体参数。

九大场景类型全覆盖：亲密对话、对抗/权力戏、揭示/发现、决策/转折点、到达/建立、追逐/动作、转变/效果、喜剧节拍、情绪低谷。

### Step 4：导演表演

> 情感变成行为——不是「悲伤」，而是「她把信折好，用双手压平，没有抬头看」。
> 演一个动作，不是一种情绪——动词化。

### Step 5：灯光与色彩作为情感

光比/方向/色温/动机光源/随转折移动的光——所有灯光决策服务于情感叙事。

### Step 6：防废片出厂测试（6 项）

1. 一句话能说清意图？
2. 每个乐器都表达同一意图？
3. 表演是可见动作（非情绪标签）？
4. 光源有世界来源？摄影机运动有理由？
5. 匹配项目风格？
6. 复杂性能否保持脸/手/logo/文字稳定？

### Step 7-9：长片脊背 + 决策程序

多片段项目需规划演进弧线（紧张上升→景别收紧→运动活跃/静止→光对比加深→结尾放松），并记录每片段的 `voice/arc_position/intention_trends/is_pattern_break`。

## 五、提示词工程（Director Formula）

### 5.1 7 槽公式

```
[主体锚点] + [可见节拍/动作] + [场景/环境] + [单一动机运镜] + [光影/风格] + [声音意图] + [约束]
```

按**优先级**排列：

| 槽位 | 用途 | 示例 |
|------|------|------|
| **主体** | 模型必须追踪的锚点 | 「原始陶瓷香水瓶，瓶身标签完全保留」 |
| **动作** | 唯一的可见变化 | 「冷凝水珠在5秒内形成并沿玻璃滑落」 |
| **场景** | 参考中未呈现的环境 | 「静谧雨夜厨房台面，浅景深」 |
| **相机** | 一个主要运动及终点 | 「从产品中景缓慢推至标签细节特写」 |
| **光影/风格** | 物理光源 + 安全视觉语言 | 「左侧暖色实用光，蓝色冷调边缘光」 |
| **音频** | 环境底噪、音效、对白或静音 | 「低环境音，最终帧玻璃轻响」 |
| **约束** | 保留项与排除项 | 「不得改变logo、形状、标签或瓶盖几何」 |

### 5.2 四种主要模式

| 模式 | 起草重点 | 常见错误 |
|------|---------|---------|
| **T2V（文生视频）** | 构建完整镜头，紧凑分层 | 单个片段中事件过多 |
| **I2V（图生视频）** | 保留可见身份，仅添加运动 | 重复描述图像导致漂移 |
| **V2V（视频生视频）** | 转移运动/相机/节奏 | 未经授权复制外貌或场景 |
| **FLF2V（首尾帧）** | 从第一帧移动到最后一帧 | 将尾帧视为模糊氛围而非固定终点 |

### 5.3 压缩规则（超长时）

删除顺序：重复风格形容词 → 通用质量词 → 参考中已可见的背景细节 → 次要相机运动 → 次要动作 → 推测性情绪标签

**保留**：约束、动作时间、角色映射。

## 六、标准操作循环（13 步）

这是处理所有复杂/多步骤/含风险请求的完整流程：

```
Step 1:  输入识别 —— 目标、制作阶段、目标平台
         ↓
Step 2:  源门 —— 平台相关问题加载 api-status 等参考
         ↓
Step 3:  专业门 —— 影视/广告/交付加载 pro-filmmaking-standards
         ↓
Step 4:  序列门 —— 判断 standalone_clip / sequence_project
         ↓
Step 5:  模式门 —— T2V / I2V / V2V / R2V / FLF2V / edit / extend
         ↓
Step 6:  能力检查 —— 规划镜头和预算时加载 capability-map
         ↓
Step 7:  参考图映射 —— 为每个参考素材分配主要角色
         ↓
Step 8:  多语言门 —— 中文/日文等提示加载对应语言参考
         ↓
Step 9:  安全门 —— IP/肖像/品牌风险路由到 copyright/filter
         ↓
Step 10: 方向 —— 确定一句话意图，让所有镜头元素服务于此
         ↓
Step 11: 提示词构建 —— 路由到 seedance-prompt 或 seedance-sequence
         ↓
Step 12: 质量检查 —— 废片检测 + 连贯性 + 导演意图一致性
         ↓
Step 13: 修复循环 —— retake-protocol 裁决 + troubleshoot 根因诊断
```

**Fast Lane（快速通道）**：单纯、单次、无安全风险的请求直接跳到 Step 10-11，跳过门槛检查。

## 七、与本项目（AIToutiao-Engine）的对接分析

### 7.1 核心差距

| 维度 | 当前状态（AIToutiao-Engine） | Seedance 2.0 Skill OS 方案 | 差距 |
|------|---------------------------|---------------------------|------|
| 视频生成模型 | Agnes Video V2.0（免费，简单Prompt） | Seedance 2.0（专业级，导演式Prompt） | **模型可替换** |
| Prompt 哲学 | 堆砌形容词（CG感/暗调/橙红烈焰） | 意图驱动（先定场景功能再选乐器） | **需要转型** |
| 分镜设计 | 无，直接写一个大Prompt | 9步导演引擎 + 13步操作循环 | **完全缺失** |
| 质量门 | 无视频质量检查 | retake-protocol + troubleshoot 体系 | **完全缺失** |
| 多片段衔接 | 手动切4段，各自独立 | seedance-sequence + continuity 体系 | **完全缺失** |
| 安全合规 | 无专门视频安全门 | copyright + filter 双重机制 | **完全缺失** |

### 7.2 可复用资产

| Seedance 2.0 组件 | 复用方式 | 优先级 |
|-------------------|---------|-------|

| **安全门** copyright/filter | 发布前合规检查 | P2 可选 |

### 7.3 对接方案建议（不实施，仅方案）

> ⚠️ 以下为建议方案，暂不实施。

**方案1：建立「视频导演Agent」**
- 创建一个新的 agent（`video-director.md`），以 Seedance 2.0 Skill OS 的导演引擎为核心思维，专门负责视频分镜设计

- 不用动代码，仅改变当前 `agnes_video_gen.py` 中 Prompt 的编写方式

- 注册字节跳动 Seedance 2.0 API
- 替换 Agnes Video 为 Seedance 2.0 作为视频生成后端
- 充分发挥 4 模态输入优势（图+文+音频参考同时输入）

## 九、结论与下一步

### 结论
1. **Seedance 2.0 Skill OS 是一个顶级的视频生成指导框架**，不是模型本身，而是一套让 AI 成为专业导演的方法论。
2. 其核心价值在于**将影视工业的专业流程注入 AI Prompt 工程**，从「堆砌形容词」升级为「意图驱动+导演思维」。
3. 与本项目当前做法（Agnes Video + 堆砌形容词式 Prompt）存在**本质差距**，引入后可大幅提升视频质量。
4. 整个系统设计严谨、覆盖全面（从创意访谈→分镜→Prompt→生成→质量检查→修复→交付），可直接作为视频生产管线的方法论基础。

### 下一步（待用户决策）
1. **立即可用**：用导演引擎的思维重写当前 4 段 Prompt，不换模型不写代码
2. **短期（1-2天）**：创建 `video-director` Agent，将导演9步决策法封装为可复用 Agent
3. **中期（3-5天）**：接入 Seedance 2.0 API，替换或并行 Agnes Video
4. **长期**：将视频生成管线（S6）纳入 AIToutiao-Engine 流水线，形成「文章→多段视频→剪映组装→抖音发布」全自动链路
