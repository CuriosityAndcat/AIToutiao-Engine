### 2.3 正则不误删推理（关键验证）

正则：`(?m)^\s*#\S+(?:\s+#\S+)*\s*$`

- `(?m)`：多行模式，`^`/`$` 匹配每行首尾。
- `^\s*`：行首可选空白。
- `#\S+`：**第一个 `#` 后必须紧跟非空白字符**（空格不匹配 `\S`）。
- `(?:\s+#\S+)*`：零或多组「空白 + `#` + 非空白」，即连续标签簇。
- `\s*$`：整行到此必须为空白或行尾。

逐类验证：

| 输入行 | 是否命中 | 推理 |
|---|---|---|
| `#无人机战争 #AI改变战场 #未来战争 #军事科技` | ✅ 命中并删除 | 首个 `#` 后 `无` 非空白；后续 ` #xxx` 簇重复匹配；行尾无残留 → 整行纯标签簇 |
| `# 正式标题`（带空格） | ❌ 不命中 | `#` 后是空格，`#\S+` 要求 `#` 后紧跟非空白，空格不满足 → 整体不匹配 |
| `## 一、起因`（章节标题） | ❌ **不误删** | 首个 `#` 后是 `#`（`##`），满足 `#\S+` 吞掉前两个 `#`；但随后是 ` 一、起因`（空格+`一`，非 `#`），`(?:\s+#\S+)*` 无法继续，且剩余 ` 一、起因` 非空，`\s*$` 不成立 → 整体不匹配 |
| `## 章节名` | ❌ 不误删 | 同上：`##` 被 `#\S+` 吞掉后，剩余 ` 章节名` 破坏尾部 `\s*$` |
| 正文正常句子（如 `你怎么看呢？`） | ❌ 不命中 | 行首非 `#`，`^\s*#` 即失败 |
| `文中一句带 #话题 的行中引用` | ❌ 不命中 | `#` 不在行首，`^\s*#` 要求行首（忽略前导空白）即为 `#`；行中 `#` 不命中 |
| `> *某引用块*` | ❌ 不命中 | 行首为 `>`，非 `#` |

> **边界补充（S1）**：极少数复合形态 `#中文句子 #标签`（行首 `#` 后接正常中文、再接空格+标签）会被整行删除——因首 token `#中文句子` 满足 `#\S+`、后续 `#标签` 满足簇匹配。但 LLM 极少产出此类（话题标签通常独占成行），现实风险极低；**行首非 `#` 的正文内容一律不受影响**。

**结论**：该正则仅命中「行首（忽略空白）为 `#` 且紧跟非空白、且整行由纯 `#xxx` 簇构成」的行，精确覆盖 LLM 自发话题标签，且**不会误删**带空格的 `# 正式标题`、`## 章节标题`、正文句子及行中 `#` 引用。

### 2.4 空行清理说明

删除整行标签后，原标签行变空行，可能产生连续 3+ 换行（如末尾 `…你怎么看呢？\n\n\n\n![封面]`）。`\n{3,}` → `\n\n` 压缩为段落间隔，保证文末为 `你怎么看呢？\n\n![封面](images/cover.png)`，与现有 `.strip()` 配合维持干净的 Markdown 段落结构。该清理只压缩 ≥3 换行，不会破坏正常单/双空行（标题与正文、章节之间的既有 `\n\n`）。

## 3. 上游清洗判断

**结论：不改动 `write_stage.py`，组装侧单点剥离已足够。**

理由：
1. **最小改动纪律**：`write_stage.py` 写出的 `content` 被多个消费方使用（原始 `generated_file`、S4 图片生成、评估器 `evaluation.py` 评分等）。在 write_stage 内清洗会改变原始稿件内容，可能干扰评估器对「LLM 原始输出完整度」的判断，并影响无图回退路径（`has_any_image=False` 时直接回退 `generated_file`）。
2. **单一规整点已覆盖**：当前「完整稿件_配图版.md」的唯一清洗入口就是 `_assemble_article_with_images` 的规整块，新增 `(c)(d)` 在该处即可彻底解决文末标签残留，无需前置。
3. **根因在展示层**：标签行本身是 LLM 风格噪声，不影响事实/结构维度评分；在最终组装（面向头条/公众号编辑器）时剥离是合理且充分的，保留原始稿件中的痕迹对调试更友好。
4. 若未来发现标签行出现在**正文中间**（非文末），再评估是否在 write_stage 增加 prompt 约束（`prompts/` 中显式禁止自发标签），本次不处理。

## 4. 验收标准

修复后，针对 S3/S4/S5 实际生成的 `完整稿件_配图版.md`：

1. **文末应为**：
   ```
   你怎么看呢？

   ![封面](images/cover.png)
   ```
   即末尾**无** `#无人机战争 #AI改变战场 #未来战争 #军事科技` 话题标签行，封面图前为单段落间隔（一个空行）。
2. **`# 正式标题`** 保留且唯一（首个），后续重复 `# ` 标题仍被去重。
3. **`## 章节标题`**（如 `## 一、起因`）全部保留，未被误删或破坏。
4. **正文正常内容**（含行中 `#` 引用、引用块 `> *…*`）不受影响。
5. 连续空行被压缩为 `\n\n`，无多余 3+ 空行块。

可由 `ui-minimal-change` 执行后，重新跑 S3→S5 流程，比对 `完整稿件_配图版.md` 文末与章节结构验证。
