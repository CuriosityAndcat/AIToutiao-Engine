# 知识库接入方案：AIToutiao-Engine

> **按 Agentic Workflow Tier 3 制定**
> 版本: v1.0 | 日期: 2026-07-12 | 状态: 待确认
>
> 约束：**零程序改动**，仅使用 CodeBuddy IDE 原生机制

---

## 一、需求理解

**用户目标**：让项目文档/知识在对话中可被 AI 自动检索和引用，提升日常开发/内容生产任务的质量和一致性。

**关键约束**：不做任何 Python/程序代码更改。

---

## 二、CodeBuddy IDE 知识库机制（关键认知）

> ⚠️ CodeBuddy **插件版**的企业 RAG `@Docs` 知识库功能与 **IDE 独立版**的机制不同。
> 本方案基于 **IDE 独立版** 可用的机制设计（零依赖、零代码）。

IDE 版提供 **四层知识注入机制**（各级均可在对话中生效）：

| 层级 | 机制 | 路径 | 触发方式 | 生命周期 |
|------|------|------|----------|----------|
| **L0 自动上下文** | Rules（规则） | `.codebuddy/rules/` | `alwaysApply: true` → 每次对话自动加载 | 持久 |
| **L1 技能包** | Skills（技能） | `.codebuddy/skills/` | AI 按对话意图**自动匹配**或手动 `使用 [技能名]` | 持久 |
| **L2 记忆偏好** | Memory（记忆） | `.codebuddy/memory/` | 自动记录 + 每次对话注入 | 累积 |
| **L3 即时引用** | @ 文件/文件夹 | 项目任意路径 | 对话中 `@文件名` 手动引用 | 单次 |

### L1 Skills 是核心知识库机制

Skills 以 `SKILL.md` 格式存储（frontmatter `name` + `description` + 正文），AI 在对话中**自动根据 description 匹配意图并加载**。这正是将项目知识 "喂给 AI" 的最有效途径。

**项目当前已有的 Skills 基础**：
- `docs/Skills/superpowers-zh-main/` — 20 个 SKILL.md（规划/审查方法论）
- `docs/Skills/agency-agents-zh-main/` — 266 中文角色 .md（角色设定库）
- `.codebuddy/rules/agentic-workflow.mdc` — Agentic Workflow 流程规则
- `AGENTS.md` — 项目地图（已自动加载）

1. 改生产逻辑 → lib/toutiao-auto-publisher/backend/
2. agent/ 是框架层，对齐 OpenAI Agents SDK + LangGraph API
3. 护栏改动需在 write_stage 新增调用点才生效

6. 改 engine_app.py UI 前查 docs/WEB_REVIEW.md

## 五、在对话工作中使用知识库

### 5.1 自动触发（推荐，零用户操作）

AI 在对话中会根据 Skill 的 `description` 自动匹配：

| 用户说... | AI 自动加载... |
|-----------|---------------|
| "写一篇关于XX的头条稿" | `toutiao-writing` Skill |
| "把这篇改成晋说风格" | `toutiao-style` Skill |
| "用前端工程师角色优化UI" | `nexus-agents` Skill |
| 任何项目相关任务 | `project-contract` Rule（始终生效） |

### 5.2 手动触发

用户也可以在对话中显式调用：

```
使用 toutiao-writing 技能，写一篇关于新能源的微头条
```

```
@docs/风格分析/包明说_风格指南.md 参考这个风格写
```

### 5.3 在 Agentic Workflow 各 Tier 中的集成

```
Tier 1 (简单) → Rules 自动生效，无需额外操作
Tier 2 (中等) → Rules + 自动匹配 Skills，Reflection 时对照 project-contract 验收标准
Tier 3 (复杂) → Rules + Skills + nexus-agents 角色分派 + @研究资产引用，Plan 阶段用 Skills 知识库作为输入
```

### 5.4 在 SAWORKFLOW 中的集成

```
Phase A (superpowers 规划) → Rules(project-contract) 提供验收标准基线
                            → Skills(toutiao-writing/toutiao-style) 提供领域上下文
Phase B (agency 执行)       → Skills(nexus-agents) 提供角色映射
                            → @docs/采集/ 提供参考素材
质量门                      → evaluation.py@75/90 (由 project-contract Rule 保障一致性)
```

## 七、风险与缓解

| 风险 | 缓解 |
|------|------|
| Skill 内容与代码漂移（如 Prompt 模块改名） | Skill 中引用文件路径而非复制内容；漂移时低成本更新 SKILL.md |
| Skill 过多导致匹配混乱 | 每个 Skill 的 `description` 精确限定触发词；KB-1 用 Rules(始终生效)而非 Skill(按需匹配) |
| project-contract 过长污染上下文 | 控制在 ~1KB 内，只放"约束"不放"说明" |
| 角色库已有 266 个 .md，不需要再建 Skill | nexus-agents Skill 只做**索引/映射**，不复制角色内容 |
